<objective>
Convert the monolithic `loom-core` into a thin re-export shim that delegates to `loom-ir-core` and `loom-container`. All existing downstream crate consumers (ingress adapters, CLI, FFI, native-melior, fixtures) continue to `use loom_core::*` without import changes. Full workspace build and test pass.

Purpose: Complete the crate split with zero downstream disruption. The shim pattern preserves backward compatibility so existing `use loom_core::...` paths resolve correctly through the re-exports.

Output: Re-export shim in `loom-core/src/lib.rs`, updated dependencies, and verified full workspace compilation.
</objective>
