<objective>
Create two new crates from the monolithic `loom-core`:

1. **`loom-ir-core`** — the independent decode IR layer: L2Core types/codec, structural/full verifier, error types, sidecar overlay model, sidecar routing, and runtime ABI. Zero Arrow dependencies, zero container/packaging knowledge.

2. **`loom-container`** — the packaging/distribution layer: container/table/layout codecs, Arrow semantic artifacts, native execution, verified lineage, artifact verifier, L1 model, L2 kernel registry, Arrow builder output, native lowering, FSST/ALP params, decode dialect. Depends on `loom-ir-core`.

The existing `loom-core` files are NOT deleted — a separate plan handles the re-export shim and downstream rewiring.

Purpose: Enforce the architectural boundary from Phase 49 — the decode IR is independent of any container. The sidecar overlay needs this separation to be real: the sidecar model lives in loom-ir-core (alongside the IR it binds to), while the container layer is a consumer.

Output: Two new crates in `crates/`, each compiling independently, with correct dependency edges verified.
</objective>
