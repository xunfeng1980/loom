  <action>
    Create `crates/loom-ir-core/src/sidecar.rs` with a host-neutral sidecar overlay data model.

    Define these types:

    1. `SidecarOverlay` — the complete sidecar that rides on a host file:
       - `ir_bytes: Vec<u8>` — encoded L2Core IR bytes (including L2IR magic + version header, as produced by `l2core_codec::encode_l2core_program`)
       - `bindings: Vec<ChunkBinding>` — per-granule content-hash bindings

    2. `ChunkBinding` — a content-hash binding for one host data granule (column chunk / fragment / row group):
       - `granule_id: String` — identifier (column name, column index, fragment id)
       - `host_data_range: (u64, u64)` — byte range (offset, length) of the host data this binding covers
       - `content_hash: String` — FNV-1a hash of the host data bytes in that range, formatted as `"l2ir:<hex>"`
       - `ir_identity: String` — the L2Core IR program hash (must match `l2core_program_hash` of the decoded ir_bytes)

    3. `SidecarCodecError` — an enum with variants:
       - `Malformed(String)` — bad format during decode
       - `Truncated` — bytes end before expected length
       - `BadHashFormat(String)` — content_hash doesn't match expected `l2ir:<hex>` format

    4. `SidecarOverlay::encode(&self) -> Vec<u8>`:
       Use the same deterministic binary encoding style as `l2core_codec.rs` (little-endian, length-prefixed):
       - `u32`: `ir_bytes.len()`
       - `ir_bytes` (raw)
       - `u16`: `bindings.len()`
       - For each ChunkBinding:
         - length-prefixed `granule_id` (u8 len + UTF-8 bytes)
         - `u64`: host_data_range.0 (offset)
         - `u64`: host_data_range.1 (length)
         - length-prefixed `content_hash` (u8 len + UTF-8 bytes)
         - length-prefixed `ir_identity` (u8 len + UTF-8 bytes)
       Encoding is deterministic: the same SidecarOverlay always produces the same bytes.
       Use `l2core_codec::write_u32`, `write_u16`, `write_u64`, `write_string` where available; define inline write functions if those helpers are private.

    5. `SidecarOverlay::decode(bytes: &[u8]) -> Result<Self, SidecarCodecError>`:
       Reverse the encoding. Handle these error cases:
       - Buffer too short for u32/u16/u64 reads → `Truncated`
       - id_len exceeds remaining bytes → `Truncated`
       - content_hash or ir_identity don't match `l2ir:<hex>` format → `BadHashFormat`
       - Any other unexpected state → `Malformed`

    6. Derive `Debug, Clone, PartialEq, Eq` for all types.
       Implement `Display` for `SidecarCodecError`.

    Do NOT:
    - Import `container_codec`, `table_codec`, `layout_codec`, or `arrow_semantic_codec` — the sidecar module must remain host-neutral and packaging-independent.
    - Implement content-hash computation over host data ranges in this task (that is Plan 50-02 routing logic).
    - Add any Parquet/Vortex/Lance-specific imports — this module is host-neutral.

    **extract_sidecar_from_parquet_metadata:**

    ```text
    pub fn extract_sidecar_from_parquet_metadata(
        metadata: &ParquetMetaData,
    ) -> Result<Option<SidecarOverlay>, SidecarCodecError>
    ```

    - Access `metadata.file_metadata().key_value_metadata()` — this returns `Option<&Vec<KeyValue>>`.
    - Iterate KeyValue entries looking for `kv.key == "loom.sidecar.v1"`.
    - If found, decode `kv.value` bytes (the value is optional `String` in Parquet Thrift — handle `None` value by continuing to next entry) via `SidecarOverlay::decode(&value_bytes)`.
    - Return `Ok(Some(overlay))` on success, `Ok(None)` if no `"loom.sidecar.v1"` key exists.
    - Map `SidecarCodecError` through directly (same error type).

    **embed_sidecar_into_parquet_metadata:**

    ```text
    pub fn embed_sidecar_into_parquet_metadata(
        metadata: &mut ParquetMetaData,
        overlay: &SidecarOverlay,
    )
    ```

    - Encode the overlay via `overlay.encode()`.
    - Access mutable `file_metadata_mut()`, get or create the key_value_metadata list.
    - Remove any existing `"loom.sidecar.v1"` entry (idempotent re-embed).
    - Push a new `KeyValue { key: "loom.sidecar.v1", value: Some(encoded_bytes_as_string) }`.
    - For each `ChunkBinding` in `overlay.bindings`, also push a per-column entry:
      `KeyValue { key: format!("loom.hash.{}", binding.granule_id), value: Some(binding.content_hash.clone()) }`.

    **Important:** The `value` field in Parquet `KeyValue` is `Option<String>`. Embed as `Some(String::from_utf8_lossy(&encoded).to_string())`. This is metadata, not binary data — the encode output is binary but survives lossless roundtrip through UTF-8 because `String::from_utf8_lossy` converts non-UTF-8 bytes to replacement characters. To ensure lossless binary embedding, base64-encode the sidecar bytes before storing in the Parquet metadata string field, and base64-decode on extraction.

    **Update:** Use base64 encoding (standard, no padding) for the `value` field to guarantee lossless binary roundtrip: `base64::Engine::encode` on embed, `base64::Engine::decode` on extract. The `loom-parquet-ingress` crate may need to add `base64` as a dependency (or use an existing project dependency).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SidecarOverlay::decode → caller | Untrusted bytes from host file metadata cross this boundary; must be fail-closed |
| embed_sidecar_into_parquet_metadata | Sidecar bytes cross into host file metadata; must be additive-only (never mutate host data) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-01 | Tampering | sidecar.rs SidecarOverlay::decode | mitigate | All decode paths check buffer bounds before each read; truncated/malformed input returns `SidecarCodecError` without panicking; length-prefixed strings enforce u8 max 255-byte IDs |
| T-50-02 | Information Disclosure | sidecar_parquet.rs embed_sidecar_into_parquet_metadata | accept | Sidecar metadata is intentionally additive in host file metadata; a Loom-unaware reader seeing unknown keys is expected behavior (strippable overlay invariant); no PII in sidecar |
| T-50-03 | Tampering | sidecar_parquet.rs base64 embed | mitigate | Base64-encode binary sidecar bytes before storing in Parquet `Option<String>` value field to guarantee lossless roundtrip; decode rejection on invalid base64 is fail-closed |
| T-50-04 | Denial of Service | sidecar.rs decode | mitigate | Bindings count is `u16` (max 65535); each granule_id is max 255 bytes; total decode time bounded by O(bindings × granule_id_len) not O(n^2); no unbounded allocation |
| T-50-05 | Elevation of Privilege | sidecar_parquet.rs embed | accept | Embed function only adds metadata entries; no host data mutation; worst case is corrupt sidecar that routing logic later rejects |
| T-50-SC | Tampering | cargo build | mitigate | No new external packages added; base64 crate already available as a transitive dependency (verify — add explicit pin if needed); slopcheck + blocking human checkpoint for any [ASSUMED]/[SUS] package |
</threat_model>
