<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external source metadata -> SourceFacts | Untrusted source metadata becomes reviewer-visible facts but not proof of safe artifact emission. |
| source adapter -> generic contract | Future adapters could overstate support or leak source SDK types into generic APIs. |
| workspace dependencies -> generic crate | Source-specific packages could accidentally enter the generic contract and spread to downstream crates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01-01 | Spoofing | SourceIdentity | mitigate | Identity/fingerprint fields are facts only; accepted report construction still requires verifier and oracle evidence. |
| T-26-01-02 | Tampering | SourceIngressReport constructors | mitigate | Constructors enforce accepted/unsupported/rejected invariants and stable tests cover invalid states. |
| T-26-01-03 | Information Disclosure | SourceDiagnostic/source_detail | mitigate | Generic diagnostics use code/path/message and optional source detail only; no credentials, object-store handles, sessions, or source SDK objects. |
| T-26-SC | Tampering | package installs/dependencies | mitigate | No package installs are planned; cargo-tree guards block source SDK and native/host dependencies in the generic crate. |
</threat_model>
