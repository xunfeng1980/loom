<objective>
Strip the Loom project to sidecar-only mode per D-01 through D-10. Delete four container-path crates, remove full-FFI and Iceberg-binding dead code, clean up loom-core re-exports and loom-fixtures container-dependent artifacts, hardcode LOOM_SIDECAR_ONLY as the sole DuckDB extension path, and update release scripts so the workspace compiles and verifies with only the sidecar execution track.

Purpose: The container path (`.loom` files, LMC1/LMP1/LMT1 codecs), full C ABI (`loom-ffi`), native `.loom` file IO (`loom-self-ingress`), and Iceberg binding stub are dead code in the sidecar repositioning model. Removing them reduces the TCB, simplifies the build, and eliminates the two-path complexity in the DuckDB extension. After this phase, the only code path is: host files → sidecar overlay → 4-gate routing → Loom-native or host-native-reader fallback.

Output: A workspace that compiles cleanly with zero references to deleted crates; a DuckDB extension that always builds sidecar-only; release scripts that no longer exercise deleted paths.
</objective>
