<objective>
Add a Loom-owned primitive Arrow/raw-buffer builder model and lower `loom.decode` builder shapes into standard-MLIR-ready buffer operations.
</objective>

<context>
The README design says output should be legal by construction through typed builders. Phase 20 must make that concrete for fixed-size primitive arrays before variable-size strings, dictionaries, nested arrays, or host-engine runtime ownership are attempted.
</context>

<threat_model>
- Threat: native lowering writes raw buffers without preserving Arrow invariants.
  Mitigation: model length, null count, validity buffer, value buffer, child arrays, and release/ownership boundary explicitly.
- Threat: variable-size or nested output sneaks into Phase 20.
  Mitigation: support only fixed-size primitive outputs and fail closed for variable-size, dictionary, RLE, FSST/string, and nested shapes.
- Threat: builder lowering becomes tied to DuckDB.
  Mitigation: keep this engine-independent and Arrow-format-oriented; host ABI is Phase 22.
</threat_model>
