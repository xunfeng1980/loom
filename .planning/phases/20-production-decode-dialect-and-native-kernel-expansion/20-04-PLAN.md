<objective>
Expand production native-lowering support beyond the bounded Int32 copy slice to primitive raw-copy and primitive multi-column table slices, with fail-closed negative coverage for unsupported kernels.
</objective>

<context>
Phase 18 already emits non-null primitive single-column and primitive struct/table artifacts. Phase 20 should make these shapes native-lowering-visible before Phase 21 widens Vortex encodings and before Phase 22 defines host execution.
</context>

<threat_model>
- Threat: kernel expansion outruns verifier evidence.
  Mitigation: every supported kernel shape must start from production support reports with discharged or not-required constraints.
- Threat: multi-column lowering loses per-column type/length facts.
  Mitigation: each column carries independent builder facts and the table plan validates shared row count.
- Threat: complex encodings enter without semantic coverage.
  Mitigation: explicitly reject dictionary, RLE, FSST, ALP, variable-size strings, nested layouts, and non-discharged constraints.
</threat_model>
