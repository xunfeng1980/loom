<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sidecar metadata -> accepted binding | Sidecar fields may be forged or stale. |
| local artifact bytes -> verifier trust | Artifact bytes must be verified before being returned. |
| local hash helper -> identity proof | SHA-256 evidence depends on command execution and parsing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-03-01 | Spoofing | manifest-only or sidecar-only success | mitigate | `bind_iceberg_ref_from_paths` requires local artifact bytes, recomputed SHA-256, verifier acceptance, source evidence, and oracle evidence per D-05/D-08. |
| T-29-03-02 | Tampering | artifact hash/content identity | mitigate | Recompute SHA-256 with adapter-local helper and fail closed on mismatch or helper failure per D-08. |
| T-29-03-03 | Tampering | verifier bypass | mitigate | Call `verify_artifact` and build `SourceArtifactVerificationSummary::accepted` only from live verifier facts per D-05. |
| T-29-03-04 | Repudiation | stale schema/snapshot/table identity | mitigate | Compare metadata and sidecar table UUID/name, schema ID, snapshot ID, and ref target before acceptance per D-03/D-08. |
| T-29-03-05 | Information Disclosure | object-store/remote artifact path | mitigate | Reject remote or credential-bearing artifact paths inherited from Plan 29-02 local-only checks per D-10. |
| T-29-03-06 | Elevation of Privilege | Phase 29 query route creep | mitigate | Accepted binding report records evidence only; no SQL, DuckDB, StarRocks, or CLI public route is added per D-01. |
| T-29-03-07 | Spoofing | source/oracle evidence artifact | mitigate | Sidecar `oracle.accepted` is descriptive only; bind reads the referenced evidence JSON and checks row count, table UUID, schema ID, snapshot ID, artifact SHA-256, source accepted status, and decoded-row/oracle accepted status before `SourceOracleEvidence::accepted` is constructed. |
</threat_model>
