<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local JSON metadata -> typed facts | Hand-authored metadata is untrusted input until typed parsing succeeds. |
| Iceberg sidecar -> Loom evidence | Sidecar strings are descriptive and may be stale or malicious. |
| local path fields -> filesystem policy | Metadata could smuggle remote catalogs, object-store locations, or credentials. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-02-01 | Tampering | local metadata JSON | mitigate | Use typed `serde_json` parsing, required identity checks, stable diagnostics, and rejected reports for malformed/missing identity per D-03/D-06. |
| T-29-02-02 | Spoofing | sidecar verifier fields | mitigate | Parser records sidecar verifier/oracle/source evidence as descriptive only; no accepted report is constructed in Plan 29-02 per D-05/D-07. |
| T-29-02-03 | Information Disclosure | remote/object-store/credential fields | mitigate | Reject or mark unsupported URI, warehouse, REST, secret, and credential markers in metadata/sidecar path fields per D-10/D-15. |
| T-29-02-04 | Repudiation | unsupported valid metadata | mitigate | Unsupported reports carry bounded facts and diagnostics but no bytes, accepted verifier evidence, or accepted oracle evidence per D-06. |
| T-29-02-05 | Tampering | SDK dependency leakage | mitigate | Focused gate continues no-default-`iceberg` dependency and public-surface scans from Plan 29-01 per D-09. |
</threat_model>
