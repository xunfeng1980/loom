<objective>
Create the Phase 21 coverage vocabulary and reader fact contract so every Vortex
shape records support, artifact emission, oracle evidence, and lowering
disposition explicitly.
</objective>

<context>
Phase 18 facts classify valid inputs as accepted/unsupported/rejected, but they
do not yet expose a coverage matrix that can distinguish reader support from
artifact emission and native lowering. Phase 21 needs that distinction before
adding nullable, chunked, dictionary, run-end, bitpack, FOR, or string coverage.
</context>

<threat_model>
- Threat: a valid Vortex shape is marked "supported" without saying whether it
  emits a Loom artifact or lowers natively.
  Mitigation: add explicit coverage and lowering disposition fields.
- Threat: Vortex API type names leak into `loom-core`.
  Mitigation: keep the coverage vocabulary in `loom-vortex-ingress` and expose
  only stable strings/enums.
- Threat: Phase 21 docs overclaim arbitrary Vortex support.
  Mitigation: maintain a finite matrix document and stable deferred diagnostics.
</threat_model>
