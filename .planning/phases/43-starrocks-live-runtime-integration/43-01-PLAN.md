# 43-01 Plan: StarRocks Runtime Evidence Contract

## Objective

Promote Phase 30's StarRocks-compatible descriptors into a typed runtime
evidence contract that can accept real StarRocks query output only when it is
bound to the same verifier-accepted artifact identity and expected result
matrix.

## Tasks

1. Add runtime evidence data structures.
   - Extend `loom-dual-query-surface` with `StarRocksRuntimeEvidence`,
     runtime status/disposition strings, and validation helpers.
   - Reuse existing `StarRocksQueryDescriptor`, binding identity, expected
     result digest, and canonical query matrix.
   - Keep skipped/missing-runtime separate from accepted live evidence.

2. Add local contract tests.
   - New test file:
     `crates/loom-dual-query-surface/tests/starrocks_runtime_contract.rs`.
   - Assert accepted runtime rows/scalars validate only when identity, query
     kind, projection, digest, and concrete output match.
   - Assert missing runtime, descriptor drift, malformed output, and result
     mismatch are not accepted.

3. Preserve dependency boundaries.
   - Do not add StarRocks/MySQL/Docker/JDBC/ODBC dependencies.
   - Update dependency-boundary tests only if their wording needs to distinguish
     runtime evidence structs from forbidden runtime client dependencies.

## Acceptance Criteria

- `cargo test -p loom-dual-query-surface --test starrocks_runtime_contract`
  passes.
- Existing Phase 30 query-surface tests still pass.
- No skipped or missing-runtime evidence can be reported as accepted live
  StarRocks evidence.

## Non-Claims

This plan defines and tests the evidence contract. It is not, by itself, proof
that a live StarRocks cluster was queried.
