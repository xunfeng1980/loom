# 43-02 Plan: Cross-Engine Equivalence and Fail-Closed Runtime Gate

## Objective

Add a focused Phase 43 gate that compares oracle/DuckDB/StarRocks runtime
evidence across the bounded matrix and records live runtime status without
turning missing runtime configuration into an accepted claim.

## Tasks

1. Add `scripts/starrocks-live-runtime-test.sh`.
   - Generate the Phase 30 accepted fixture bundle and descriptors.
   - Run existing Rust contract tests and DuckDB evidence tests.
   - When explicit StarRocks env/client inputs are present, run the bounded SQL
     matrix against the live runtime and validate rows/scalars.
   - When live inputs are absent, emit a clear missing-runtime diagnostic and
     leave live acceptance pending unless a strict live flag is requested.

2. Add runtime report.
   - Create `43-STARROCKS-RUNTIME-REPORT.md`.
   - Record the supported query matrix, env requirements, exact acceptance
     rules, fail-closed cases, and current local live-runtime status.
   - State whether live runtime evidence has actually been collected in this
     environment.

3. Add fail-closed checks.
   - Gate markers must prove unsupported shapes, drifted descriptors, and
     mismatched runtime output cannot become accepted evidence.
   - Strict mode must fail if live runtime evidence is required but missing.

## Acceptance Criteria

- `bash scripts/starrocks-live-runtime-test.sh` passes in local contract mode.
- `LOOM_REQUIRE_STARROCKS_LIVE=1 bash scripts/starrocks-live-runtime-test.sh`
  fails closed when no live runtime env/client is available.
- If live StarRocks env/client inputs are available, the script validates
  ordered rows, predicate rows, count, and sum against the accepted matrix.

## Non-Claims

Local contract mode is not live StarRocks evidence. The report and script must
say that plainly.
