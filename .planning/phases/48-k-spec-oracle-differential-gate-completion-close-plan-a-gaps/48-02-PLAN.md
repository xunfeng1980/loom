<objective>
Add an in-process per-shape native-route disable registry to jit.rs. On a native↔K trace divergence (the hard-fail divergence outcome from plan 48-01), record the shape's schema_fingerprint as disabled, fail-closed for that run, and ensure subsequent route requests for that shape short-circuit to interpreter fallback before JIT and before any krun invocation. Disabled/divergent shapes must never seed the runtime cache or replay evidence, consistent with Phase 43.2 admission discipline.

Purpose: Plan-A adjudication requires a divergent shape to be disabled for the process lifetime, not merely diagnosed. This closes gap #2. The registry composes with (does not bypass) Phase 43.2 cache/replay admission: disabled shapes can never become NativeCandidate, which is the precondition for cacheability.

Output: A module-level `OnceLock<Mutex<HashSet<String>>>` registry with `is_shape_disabled` / `disable_shape` helpers and a `#[cfg(test)] reset_disabled_shapes`, a pre-check + post-validation disable hook in `execute_arrow_semantic_codegen_production_route_inner`, a `NativeShapeDisabled` route diagnostic, and a shape-disable test file.
</objective>
