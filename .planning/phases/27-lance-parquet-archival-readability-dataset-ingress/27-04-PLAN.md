<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| supported source batches -> Loom artifact bytes | Source rows become `LMC1` only after classification, conversion, verification, and oracle checks. |
| source oracle -> evidence | Source-native/Arrow scan output is evidence, not a decode bypass. |
| legacy fixture manifest -> archival claim | Legacy metadata can drift; tests must verify paired Loom bytes and rewrite current output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-04-01 | Tampering | Parquet/Lance emit helpers | mitigate | Call `verify_artifact` before `SourceIngressReport::accepted`; return bytes only in `SourceIngressAcceptedArtifact`. |
| T-27-04-02 | Repudiation | Oracle evidence | mitigate | Record `SourceOracleEvidence` strategy, row count, null checks, notes, and decoded row equality tests. |
| T-27-04-03 | Tampering | Unsupported/rejected paths | mitigate | Unsupported and rejected results return `Err(SourceIngressReport)` with no artifact bytes. |
| T-27-04-04 | Information Disclosure | Oracle diagnostics | mitigate | Diagnostics avoid credentials, SDK handles, scanner state, and object-store configuration. |
| T-27-04-05 | Tampering | Legacy fixture drift | mitigate | Manifests record generator/version or gap, expected rows, paired artifact provenance, and current rewrite evidence. |
| T-27-04-06 | Elevation of Privilege | Source semantics | mitigate | Canonical raw/table emission is reported only for supported primitive/table rows; no Lance index, Parquet footer, predicate, split, or native-kernel semantics are claimed. |
</threat_model>
