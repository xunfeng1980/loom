<decision_trace>
D-01: Both Lance and Parquet are in scope, each as a minimal local-file adapter.
D-02: Accepted emission is limited to non-null primitive Int32, Int64, Float32, and Float64 single-column and simple table shapes representable by `LMP1` or `LMT1`.
D-03: Facts may describe Lance schema/version/fragments and Parquet schema/row groups/page-adjacent metadata, but only as source-neutral summaries.
D-04: Source-specific crates own Lance and Parquet SDK dependencies.
D-05: `loom-core`, `loom-ffi`, `loom-source-ingress`, DuckDB extension code, CLI public route surface, and public headers remain free of Lance/Parquet SDK dependencies and APIs.
D-06: Generic public types must not expose Lance/Parquet SDK objects, dataset handles, readers, credentials, Arrow stream ownership handles, or object-store state.
D-07: Each source family needs current-version read/write/verify evidence and legacy-file-with-Loom readability evidence where feasible.
D-08: Loom artifacts are paired with source facts and evidence, not embedded into Lance manifests or Parquet footers.
D-09: If old-version source writer tooling is unavailable without brittle builds, use a paired legacy-style fixture manifest and record the gap.
D-10: Parquet oracle uses Arrow scan output; Lance oracle uses Lance-native or Arrow-compatible scan output.
D-11: Accepted reports require `SourceOracleEvidence` and accepted artifact verification.
D-12: Equivalence compares verified Loom artifact rows against source-native or Arrow oracle rows for the supported slice.
D-13: The phase writes `27-ARCHIVAL-READABILITY-REPORT.md` and adds a bounded `scripts/lance-parquet-ingress-test.sh` release gate.
D-14: Use researched exact package defaults unless a minimal compile-valid feature adjustment is required and documented.
D-15: Lance adapter APIs may be async-only with `#[tokio::test]`; no public blocking wrapper.
D-16: Legacy fixture default is one tiny checked-in fixture per family generated by the oldest feasible stable/legal version, or a paired legacy-style manifest with the gap recorded.
</decision_trace>
