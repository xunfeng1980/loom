<decision_trace>
D-01: Use a narrow adapter-local Iceberg binding crate/module; do not add public SQL, public C ABI, DuckDB routes, or StarRocks integration.
D-02: Bind Loom artifacts through explicit sidecar/reference metadata rather than embedding Loom bytes into Iceberg manifests or Parquet footers.
D-03: Represent Iceberg identity with bounded fields: table UUID/name, snapshot ID, schema ID, metadata/manifest locations, and content/hash references.
D-04: Keep `loom-source-ingress` source-neutral; Iceberg vocabulary belongs only in the adapter/report boundary.
D-05: Accepted entries require an existing verifier-accepted Loom artifact; verifier summaries are evidence, not unchecked trust tokens.
D-06: Preserve Phase 26/27 accepted/unsupported/rejected semantics.
D-07: Treat Iceberg schema/snapshot facts as descriptive until matched to verifier-accepted Loom artifact identity and source evidence.
D-08: Fail closed on stale snapshot/schema/artifact hash mismatch and manifest-only success claims.
D-09: Do not add the `iceberg` crate by default because current `iceberg 0.9.1` pulls Arrow/Parquet 57.1 into this Arrow/Parquet 58.3 workspace.
D-10: Local metadata fixtures are in scope; remote catalogs, REST auth, object-store credentials, table commits, branch/tag mutation, and snapshot lifecycle management are out of scope.
D-11: Do not broaden Lance/Parquet compatibility, native kernels, predicate pushdown, split execution, nullable/nested coverage, or full Vortex compatibility.
D-12: Produce `28-ICEBERG-BINDING-REPORT.md`.
D-13: Produce `scripts/iceberg-binding-test.sh` and wire it into `scripts/mvp0-verify.sh` only after the focused gate passes.
D-14: Main release order must run Phase 28 after Phase 27 and before DuckDB smoke.
D-15: Include negative tests for public-surface creep, object-store/catalog credential creep, unchecked manifest-only success, stale snapshot/schema/hash mismatch, and SDK leakage.
D-16: Record current-phase tradeoffs in the final report.
D-17: Planner choice: use `loom-iceberg-binding`, standalone local sidecar JSON, hand-authored metadata fixtures, and `SourceOracleStrategy::DecodedRowFixture`.
</decision_trace>
