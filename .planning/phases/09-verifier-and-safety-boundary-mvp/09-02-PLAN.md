<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Rust public decode helper input | In-process callers can construct `LayoutDescription` directly. |
| FFI payload input | C/C++ host supplies untrusted bytes to `loom_decode`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-09-05 | Tampering | direct Rust decode helper | mitigate | Call verifier before builder/decode output. |
| T-09-06 | Tampering | FFI payload bytes | mitigate | Parse and verify before Arrow C Data export. |
| T-09-07 | Denial of Service | duplicated verifier/decode logic | mitigate | Keep data-dependent checks in decode-time authoritative code to reduce drift. |

No high open threats remain if malformed verifier failures return decode errors before Arrow export.
</threat_model>
