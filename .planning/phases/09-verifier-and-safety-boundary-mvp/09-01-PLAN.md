<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Untrusted payload/descriptor -> verifier | User-controlled layout/table descriptions are inspected before decode. |
| Verifier -> decode | A passing report is not a formal proof; decode-time typed checks remain authoritative for deeper data-dependent invariants. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-09-01 | Tampering | verifier diagnostics | mitigate | Include stable code and path so malformed structure is inspectable and not opaque. |
| T-09-02 | Spoofing | unknown L2 kernels | mitigate | Verify `KernelEscape.kernel_id` against `L2KernelRegistry` before decode. |
| T-09-03 | Denial of Service | count arithmetic | mitigate | Use checked arithmetic for byte/count validations. |
| T-09-04 | Repudiation | recursive layout failures | mitigate | Path-aware diagnostics identify the failing node. |

No high open threats remain if diagnostics are typed and count arithmetic is checked.
</threat_model>
