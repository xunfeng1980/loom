<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI input file -> verifier display | Reviewer supplies a payload/descriptor path to inspect. |
| Release gate -> malformed fixtures | The gate intentionally feeds malformed data to prove fail-closed behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-09-08 | Repudiation | CLI verifier output | mitigate | Print status and diagnostics by default in `loom inspect`. |
| T-09-09 | Tampering | negative test fixtures | mitigate | Generate deterministic temp inputs and assert exact failure behavior. |
| T-09-10 | Denial of Service | release gate drift | mitigate | Invoke negative gate from `scripts/mvp0-verify.sh`. |

No high open threats remain if inspect output and release gate checks are deterministic.
</threat_model>
