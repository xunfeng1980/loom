<objective>
Add an explicit Phase 17 adapter for verifying an `LMC1` artifact together with an associated `L2CoreProgram`, then fuse accepted `VerifiedArtifactFacts`, proof IDs, and constraint status into the unified artifact report.
</objective>

<context>
Phase 13 has a Rust `L2CoreProgram` data model and `verify_l2_core`, but no stable external L2Core artifact codec. Phase 17 should not invent that codec. It should provide an explicit adapter so later lowering and tests can consume one artifact report while the codec remains deferred.
</context>

<threat_model>
- Threat: standalone copied `VerifiedArtifactFacts` are treated as trusted.
  Mitigation: facts are only copied out of the `FullVerificationReport` returned by `verify_l2_core` in the same adapter call.
- Threat: L2Core rejection still leaves artifact-level accepted facts.
  Mitigation: rejected L2Core diagnostics make the unified report rejected and hide facts.
- Threat: Phase 17 claims solver discharge without a solver.
  Mitigation: non-empty constraint IDs set `ConstraintDischargeStatus::CollectedOnly`.
</threat_model>
