<objective>
Implement the first executable unified verifier entrypoint for `LMC1` bytes by decoding the container, extracting container/manifest facts, reusing the existing L1 structural verifier, and returning a structured `ArtifactVerificationReport`.
</objective>

<context>
This plan connects Phase 11 container artifacts and Phase 9 structural verification into the Phase 17 report model. It intentionally does not parse or verify external L2Core artifact sections yet.
</context>

<threat_model>
- Threat: `verify_artifact` bypasses existing structural checks.
  Mitigation: it must call `verify_container(bytes, registry)` for current `LMC1` payloads.
- Threat: unknown or unsupported artifact combinations look accepted.
  Mitigation: unsupported combinations return `ArtifactVerificationStatus::Unsupported` with no accepted facts.
- Threat: malformed bytes panic during verification.
  Mitigation: malformed containers become `Container` stage diagnostics.
</threat_model>
