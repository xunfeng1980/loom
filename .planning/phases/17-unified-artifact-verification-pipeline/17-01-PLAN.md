<objective>
Define the unified artifact verifier contract and Rust report model that Phase 17 will use to connect container/L1 verification, optional L2Core verification, constraint status, facts fusion, and lowering readiness without changing existing verifier behavior yet.
</objective>

<context>
Phase 17 exists because `verifier.rs` and `full_verifier.rs` currently form two separate verification lines. This plan creates the shared vocabulary and invariants before wiring any existing verifier into the new pipeline.
</context>

<threat_model>
- Threat: rejected or unsupported artifacts expose trusted facts.
  Mitigation: `ArtifactVerificationReport::facts()` returns `Some` only when status is `Accepted`.
- Threat: diagnostic stages become ambiguous after multiple verifiers are joined.
  Mitigation: every diagnostic carries an `ArtifactVerificationStage`.
- Threat: Phase 17 accidentally claims solver discharge.
  Mitigation: represent solver state with `ConstraintDischargeStatus::CollectedOnly` and do not add Z3/cvc5 dependencies.
</threat_model>
