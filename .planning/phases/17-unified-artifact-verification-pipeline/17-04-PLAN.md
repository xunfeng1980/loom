<objective>
Make lowering readiness an explicit fail-closed decision over the unified artifact report, expose the report through the CLI, and add a release-gate script for Phase 17 verifier pipeline coverage.
</objective>

<context>
Phase 14/16 lowering currently consumes `FullVerificationReport` directly. Phase 17 should introduce a unified artifact-level readiness gate without creating new MLIR, LLVM, JIT, or native artifacts.
</context>

<threat_model>
- Threat: valid structural artifacts are mistaken for native-lowering ready artifacts.
  Mitigation: lowering readiness is false unless L2Core facts and support predicates accept the target.
- Threat: CLI hides unsupported versus rejected outcomes.
  Mitigation: CLI prints status, stage-addressed diagnostics, facts presence, constraint status, and lowering readiness.
- Threat: release gates miss the new verifier pipeline.
  Mitigation: add `scripts/artifact-verifier-test.sh` and call it from `scripts/mvp0-verify.sh`.
</threat_model>
