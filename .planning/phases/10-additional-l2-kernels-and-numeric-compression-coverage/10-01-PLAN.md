<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Untrusted ALP params -> `AlpParams::decode` | Malformed bytes must be rejected without panics or unbounded allocation. |
| Kernel registry -> ALP decode | Stable ids must remain append-only and type-checked. |
| Float params -> Arrow output | Decoded arrays must have the declared Float32/Float64 type and correct nulls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-10-01 | Tampering | kernel id registry | mitigate | Append ALP at id `1`; add tests proving FSST remains id `0`. |
| T-10-02 | Tampering | ALP params | mitigate | Magic/version/length/type/validity validation in `AlpParams::decode`. |
| T-10-03 | Denial of Service | row count and vector lengths | mitigate | Checked length conversions and row-count equality before allocation-sensitive decode. |
| T-10-04 | Spoofing | output data type | mitigate | Verifier checks `AlpParams.output_type == LayoutDescription.data_type`. |

No high open threats remain if params decode is total and verifier rejects type/kernel mismatches before decode.
</threat_model>
