## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQL file path to bind | Local path bytes are read into DuckDB and forwarded to Rust verification. |
| Bind plan to global init | Opaque runtime plan handles cross DuckDB lifecycle callbacks. |
| Global init to backend prepare | Native backend may run only after runtime route approval. |
| Error/cancel path to Arrow RAII | Partially decoded Arrow arrays must remain owned by scan state until teardown. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-03-01 | Tampering | `LoomBind` native route | mitigate | Call Rust runtime planning helper; no C++ native eligibility switch. |
| T-24-03-02 | Repudiation | fallback route | mitigate | Store route decision/cache and surface stable diagnostic code/path in errors and tests. |
| T-24-03-03 | Denial of Service | Arrow release on route error | mitigate | Keep existing `LoomScanState` RAII release and transfer each successful Arrow pair into state immediately. |
