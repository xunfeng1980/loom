<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| C++ to internal Rust C ABI | Raw pointers, lengths, and opaque handles cross the unsafe FFI boundary. |
| Internal ABI to public ABI surface | Build tooling could accidentally expose private route controls in generated public headers. |
| Prepared handle to C++ native buffers | Borrowed native buffer pointers are valid only while the prepared handle lives. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-02-01 | Denial of Service | `loom_duckdb_*` wrappers | mitigate | Null guards before pointer use and `catch_unwind` around every wrapper body. |
| T-24-02-02 | Elevation of Privilege | test-only controls | mitigate | Keep controls in `loom_duckdb_internal.h`, not public SQL or `loom.h`, and label them as internal. |
| T-24-02-03 | Information Disclosure | diagnostic strings | accept | Diagnostics contain code/path/message for local artifacts only; no remote secrets or auth state exist. |
| T-24-02-04 | Tampering | public ABI freeze creep | mitigate | Grep gate `loom.h` for no `loom_duckdb_`, no `LoomDuckDb`, and no `ArrowArrayStream`. |
| T-24-02-SC | Tampering | npm/pip/cargo installs | accept | No new registry package installs are planned. |
</threat_model>
