<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DuckDB host to Rust planning bridge | Untrusted local artifact bytes and host projection enter verifier/runtime policy. |
| Rust runtime policy to native backend | Only native-candidate plans with cache identity and lowering facts may enter backend prepare. |
| Native output to DuckDB fill path | Native bytes are correctness-sensitive and must be compared before use. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01-01 | Tampering | `plan_duckdb_runtime` | mitigate | Verify artifact bytes and call `decide_runtime_execution`; do not use host-provided native claims. |
| T-24-01-02 | Repudiation | route diagnostics | mitigate | Preserve runtime/backend code, path, and message in `DuckDbRuntimeDiagnostic`. |
| T-24-01-03 | Denial of Service | native-output-mismatch | mitigate | Treat `NativeOutputMismatch` as fail-closed and return no native buffers. |
| T-24-01-04 | Denial of Service | cancellation | mitigate | Convert cancellation to backend cancellation diagnostics and no partial output. |
| T-24-01-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are planned; only workspace path dependency wiring is used. |
</threat_model>
