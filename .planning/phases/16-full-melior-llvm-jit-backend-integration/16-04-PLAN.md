<objective>
Execute the verified bounded Int32 copy slice through MLIR ExecutionEngine/JIT when a compatible `melior`/LLVM toolchain is present, and compare the native result against Phase 14's Rust typed primitive reference output.
</objective>

<context>
This is the first Phase 16 plan that attempts actual JIT execution. It is intentionally limited to typed primitive buffers and must not introduce Arrow raw-buffer writes, DuckDB execution, a custom Loom dialect, vectorization, or multi-column native lowering.
</context>

<threat_model>
- Threat: native execution writes outside output bounds.
  Mitigation: allocate output length from `VerifiedArtifactFacts.row_count_bound` and reject before invocation when input length is shorter than that bound.
- Threat: JIT invokes the wrong symbol or ABI.
  Mitigation: use one stable entry symbol, assert lookup success, and record `jit-symbol-missing` when absent.
- Threat: native result differs from interpreter/reference path but still reports success.
  Mitigation: compare output exactly against `execute_supported_copy_i32` and emit `native-output-mismatch` on any length or value mismatch.
</threat_model>
