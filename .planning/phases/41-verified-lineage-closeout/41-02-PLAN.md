# Phase 41-02 Plan: Verified-Lineage Record and Docs

**Status:** Planned
**Date:** 2026-06-09
**Requirements:** LINEAGE-12

## Goal

Add an artifact-facing verified-lineage record API that names safety evidence
layers and TCB assumptions for accepted artifacts, plus docs that define the
claim boundary precisely.

## Tasks

1. Add a `verified_lineage` module in `loom-core`.
2. Define:
   - `VerifiedLineageRecord`;
   - `VerifiedLineageEvidenceLayer`;
   - `VerifiedLineageTcbAssumption`;
   - stable diagnostic/status fields for rejected/unsupported inputs.
3. Build records from:
   - accepted `ArtifactVerificationReport` facts;
   - `ConstraintDischargeStatus` / solver report presence;
   - Lean soundness gate identity;
   - Lean/Rust, model/Rust, and native/model validation gate identities;
   - optional `NativeArrowSemanticModelValidationReport`.
4. Ensure rejected/unsupported verifier reports do not produce a positive
   verified-lineage record.
5. Add focused tests covering:
   - accepted artifact record includes structural verifier, Bitwuzla discharge
     when present, Lean soundness, differential validation, and TCB rows;
   - native/model validation success adds native/model evidence;
   - divergent/unsupported native validation is recorded as non-validated or
     fails closed;
   - no correctness or verified-compilation claim appears in the record.
6. Update `README.md`, `README-zh.md`, requirements, roadmap, project/state, and
   add `41-02-SUMMARY.md`.
