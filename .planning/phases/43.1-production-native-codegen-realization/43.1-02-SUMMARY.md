# Phase 43.1-02 Summary: Production MLIR/JIT Arrow Buffer Output

**Status:** Complete
**Date:** 2026-06-09
**Plan:** `43.1-02-PLAN.md`

## What Changed

- Added a dedicated Arrow semantic production JIT entry:
  `loom_arrow_semantic_codegen_buffers`.
- Added `execute_arrow_semantic_codegen_jit` in `loom-native-melior`.
- Generated MLIR over the Phase 35 supported matrix:
  - Int32, Int64, Float32, and Float64 value buffers use typed memrefs.
  - Boolean value buffers use byte-addressed bitmap memrefs.
  - Nullable validity buffers use byte-addressed bitmap memrefs.
- The real melior `ExecutionEngine` writes every output buffer. Positive tests
  compare those JIT-produced buffers against the real Arrow input buffers.
- The non-feature path returns `JitUnavailable` and cannot masquerade as a
  successful native-codegen run.

## Verification

All checks passed:

```sh
cargo test -p loom-native-melior --test production_arrow_semantic_jit
cargo test -p loom-native-melior --features melior --test production_arrow_semantic_jit
cargo test -p loom-native-melior --test production_backend_jit
```

## Evidence

- The focused positive test invokes melior `ExecutionEngine` with the `melior`
  feature enabled.
- The test covers 9-row nullable data so Boolean and validity bitmap output is
  not limited to byte-aligned row counts.
- `reference_zeroed_value_bytes` is not used by the production Arrow semantic
  codegen path.

## Non-Claims

- This is not verified compilation.
- This does not add Utf8, logical, nested, multi-batch, or StarRocks runtime
  support.
