# Phase 43.1 Context: Production Native Codegen Realization

## Why This Phase Exists

Phase 35 proved a bounded engine-neutral native Arrow semantic execution slice,
but the implemented positive path still copies decoded Arrow values through
Rust/Arrow builders. That is useful execution evidence, but it is not production
native codegen.

Phase 43.1 is the 44-pre / 44A bridge before ABI freeze. It must replace the
placeholder/native-copy claim surface with real MLIR/LLVM/JIT/native backend
output for the Phase 35 supported Arrow semantic matrix, then require Phase 40
native/model validation before any positive runtime/cache admission.

## Scope

In scope:

- Default `LMC2(LMA1)` Arrow semantic artifacts.
- Explicit direct `LMA1` bridge artifacts, retained as regression evidence.
- Exactly one Arrow RecordBatch.
- Nullable fixed-width primitive matrix from Phase 35:
  - Boolean
  - Int32
  - Int64
  - Float32
  - Float64
- Real native output buffers produced by the MLIR/LLVM/JIT/native backend path.
- Phase 40 model validation before native admission.
- Cache identity that binds artifact, verifier, backend, toolchain, pipeline,
  target/layout, and trace fingerprints.

Out of scope:

- Verified compilation of MLIR/LLVM.
- Utf8, logical, nested, multi-batch, dictionary, list, struct, and arbitrary
  Arrow matrices beyond Phase 35.
- Live StarRocks runtime evidence.
- Freezing the public runtime ABI; Phase 44 owns that.

## Current Implementation Snapshot

- `crates/loom-core/src/native_arrow_semantic.rs` verifies `LMC2` / `LMA1`
  Arrow semantic artifacts and emits equivalent RecordBatch output by copying
  supported columns through Arrow builders.
- Phase 40 already validates a supplied native RecordBatch against model trace
  and decoded Arrow value equivalence.
- `crates/loom-native-melior/src/jit.rs` has a real melior ExecutionEngine path
  for raw primitive value-buffer copy, but it only handles Int32, Int64, Float32,
  and Float64 value buffers.
- `crates/loom-core/src/arrow_buffer_lowering.rs` still rejects nullable output
  and lacks Boolean buffer lowering.
- `crates/loom-core/src/production_native_lowering.rs` only supports
  `LMP1 layout` / `LMT1 table` verifier facts, not `Arrow semantic payload`.

## Red Lines

- `reference_zeroed_value_bytes` must not appear in positive Phase 43.1 native
  codegen evidence.
- Rust/Arrow builder copy output must not satisfy CODEGEN-01.
- `LOOM_ALLOW_NATIVE_TOOL_SKIP=1` may keep broad gates runnable, but a skipped
  toolchain cannot be counted as positive production native codegen realization.
- Interpreter fallback may be accepted runtime policy, but it cannot seed native
  cache identity or mark native codegen success.
- Phase 40 validation is admission control, not a post-hoc note.
