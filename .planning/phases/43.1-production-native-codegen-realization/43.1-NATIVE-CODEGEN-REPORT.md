# Phase 43.1 Native Codegen Report

**Status:** Complete
**Date:** 2026-06-09

## Claim

Phase 43.1 realizes production native codegen for the Phase 35 Arrow semantic
matrix. Supported verifier-accepted `LMC2(LMA1)` and explicit direct `LMA1`
one-batch nullable fixed-width primitive artifacts can now produce Arrow
buffers and RecordBatch output through the real MLIR/LLVM/JIT/native backend
path.

## Supported Matrix

| Artifact | Batch shape | Types | Nullability | Native output |
|---|---|---|---|---|
| `LMC2(LMA1)` | exactly one RecordBatch | Boolean, Int32, Int64, Float32, Float64 | nullable or non-null | MLIR ExecutionEngine-produced value and validity buffers |
| direct `LMA1` | exactly one RecordBatch | Boolean, Int32, Int64, Float32, Float64 | nullable or non-null | same, retained as explicit bridge evidence |

Unsupported shapes fail closed before positive codegen evidence: Utf8, logical,
nested, multi-batch, malformed, verifier-rejected, or non-Arrow semantic
payloads.

## Backend Path

The production Arrow semantic codegen path is:

1. Verify artifact bytes with the artifact verifier.
2. Decode exactly one Arrow semantic RecordBatch.
3. Extract real Arrow value and validity buffers.
4. Generate MLIR for those buffers:
   - typed memrefs for Int32, Int64, Float32, and Float64 value buffers;
   - byte memrefs for Boolean value bitmaps;
   - byte memrefs for nullable validity bitmaps.
5. Lower through the melior LLVM pipeline and invoke the ExecutionEngine.
6. Rebuild Arrow arrays from JIT-produced buffers using safe Arrow builders.
7. Run Phase 40 native/model validation.
8. Admit runtime/cache identity only after validation succeeds.

## Validation and Cache Identity

Runtime/cache identity includes:

- artifact digest;
- artifact kind and payload kind;
- schema fingerprint;
- backend identity;
- pipeline/entry identity;
- row and column counts;
- Phase 40 reference/native trace fingerprints;
- runtime policy, projection, predicate, and split facts.

Divergent native output fails closed under strict policy. It may take an
explicit interpreter fallback route only when fallback policy allows it, and it
cannot seed native cache identity.

## TCB

This phase is not verified compilation. The following remain trusted components:

- Rust compiler and standard library;
- arrow-rs buffer/array construction;
- melior bindings;
- MLIR/LLVM lowering and ExecutionEngine;
- host CPU behavior.

accepted as positive evidence.
