# Phase 43.1-03 Summary: Phase 40 Admission and Cache Identity

**Status:** Complete
**Date:** 2026-06-09
**Plan:** `43.1-03-PLAN.md`

## What Changed

- Added `NativeArrowSemanticCodegenOutputColumn` and
  `NativeArrowSemanticCodegenExecutionReport` in `loom-core`.
- Added `validate_native_arrow_semantic_codegen_output`, which rebuilds Arrow
  arrays from JIT-produced value/validity buffers using safe Arrow buffer
  construction.
- The rebuilt RecordBatch is admitted only through existing Phase 40 model
  validation:
  - reference executor trace equality;
  - native trace equality;
  - decoded Arrow value equivalence.
- Added runtime/cache helpers:
  - `decide_validated_native_arrow_semantic_codegen_runtime`
  - `validated_native_arrow_semantic_codegen_runtime_cache_key`
- Cache identity now binds artifact digest, schema/verifier facts, production
  backend identity, Phase 40 trace fingerprints, row count, and column count.

## Verification

All checks passed:

```sh
cargo test -p loom-core --test native_arrow_semantic_codegen
cargo test -p loom-core --test native_arrow_semantic
cargo test -p loom-native-melior --features melior --test production_arrow_semantic_jit
```

## Evidence

- Validated production-codegen output becomes a `NativeCandidate`.
- Divergent codegen output fails closed under strict policy and can only choose
  explicit interpreter fallback when fallback policy allows it.
- Divergent output cannot seed a native runtime cache key.

## Non-Claims

- Phase 40 remains per-run translation validation, not verified compilation.
- MLIR/LLVM/melior/ExecutionEngine and CPU behavior remain TCB assumptions.
