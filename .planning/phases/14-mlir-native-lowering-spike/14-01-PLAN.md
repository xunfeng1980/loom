## Trust Boundaries

| Boundary | Description |
|---|---|
| verifier -> lowering | Lowering must not proceed without accepted report and facts. |
| support predicate -> emitter | Unsupported statements must reject before any MLIR/native artifact exists. |
| Rust workspace -> external toolchain | Phase 14 plan 01 must not introduce mandatory MLIR/LLVM dependencies. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-14-01 | Spoofing | verifier facts | mitigate | Lowering API accepts report plus program, not standalone facts. |
| T-14-02 | Tampering | unsupported program shapes | mitigate | Stable support diagnostics and negative tests for unsupported shapes. |
| T-14-03 | Elevation of privilege | native backend | mitigate | No MLIR/native artifact is emitted in 14-01; support gate only. |

No high open threats remain if support checks fail closed and tests cover rejected shapes.
</threat_model>
