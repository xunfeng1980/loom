# 42-01 Plan: Vortex Verified/Native Disposition Matrix

## Objective

Add a Phase 42 Vortex row set that widens the reviewer-visible matrix while
preserving the difference between original Vortex shape, emitted
`LMC2(LMA1)`/canonical shape, verified-lineage evidence, and native eligibility.

## Tasks

1. Add Phase 42 row helpers for Vortex coverage.
   - Read `crates/loom-vortex-ingress/src/lib.rs` and Phase 21/28 matrix code.
   - Reuse existing `VortexEncodingCoverage` and
     `VortexSemanticCompatibilityRow` where possible.
   - Add stable row IDs for primitive, UTF-8, struct/table, nullable deferred,
     dictionary/RLE, bitpack, FOR, and canonical raw rows.

2. Add focused Vortex tests.
   - New test file: `crates/loom-vortex-ingress/tests/phase42_vortex_coverage_matrix.rs`.
   - Assert accepted canonicalized rows have verifier/oracle evidence and are
     not labeled as original structured native support.
   - Assert native-supported rows require `native-execution-engine-output` or
     the Phase 35/40 native model evidence marker.
   - Assert deferred nullable/string/structured rows fail closed with stable
     disposition strings.

3. Start the living matrix.
   - Create `42-COVERAGE-MATRIX.md`.
   - Record Vortex rows, disposition, oracle/verifier evidence, native class,
     and residual deferrals.

## Acceptance Criteria

- `cargo test -p loom-vortex-ingress --test phase42_vortex_coverage_matrix`
  passes.
- `bash scripts/vortex-semantic-compatibility-test.sh` still passes.
- Vortex rows do not claim native support for original dictionary/RLE/bitpack/FOR
  encodings unless native evidence names that exact emitted shape.

## Non-Claims

Canonical raw Vortex rows are verified value bridges, not preservation of the
original encoding semantics and not automatic native support for that encoding.
