<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| vortex_reader (Plan 02) -> loom-core LayoutNode | LayoutNode fields (bit_width, offset, count, values_buf length) are the only input loom-core decodes; treated as potentially malformed even though Phase-3 sources are trusted fixtures |
| loom-core decode -> loom-ffi catch_unwind (existing) | A decode panic would cross the FFI frame; loom-core must return typed errors, not panic, so the existing catch_unwind never has to convert a panic into Panicked for normal malformed input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | bitpack::unpack_all buffer indexing | mitigate | Validate `packed.len() >= num_chunks * elems_per_chunk * (t_bits/8)` before any indexing; return `BufferTooShort` (Task 2) — prevents out-of-bounds reads on a short packed buffer |
| T-03-02 | Denial of Service | index/shift/size arithmetic in unpack_all | mitigate | Use `div_ceil` and `checked_*` for chunk-count and byte-offset math; reject t_bits not in {32,64} and bit_width > t_bits via typed error (Task 2) — no overflow panic |
| T-03-03 | Denial of Service | synthesized_read_loop on unimplemented / malformed LayoutNode | mitigate | Dictionary/RunEnd/KernelEscape arms and bounds failures return `LoomDecodeError` (typed), never `todo!()`/`panic!()` (Tasks 1-3) — keeps the eventual catch_unwind boundary clean |

No high open threats. Phase 3 is a pure-Rust in-process library: no network I/O, no file I/O, no untrusted deserialization. Untrusted-input hardening proper is the deferred verifier (out of scope).
</threat_model>
