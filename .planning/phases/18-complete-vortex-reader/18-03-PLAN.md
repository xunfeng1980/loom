<objective>
Expand real `.vortex` -> `LMC1` emission from the Phase 15 non-null Int32 slice into an explicit, verifier-backed single-column support matrix while keeping unsupported shapes fail-closed.
</objective>

<context>
Phase 18 should not support every Vortex encoding, but it should escape the one-shape Phase 15 proof. The first expansion should target single-column shapes that Loom already knows how to represent and verify: primitive/bool raw-compatible outputs and explicit null handling where evidence is available.
</context>

<threat_model>
- Threat: conversion silently uses Vortex scan semantics as implementation.
  Mitigation: every supported shape must map to a Loom `LayoutDescription` / `LMC1` payload and pass `verify_artifact`; Vortex scan is used only for oracle comparison.
- Threat: support status overclaims encoded-layout extraction.
  Mitigation: accepted shapes are limited to explicit dtype/layout/encoding predicates; other shapes return unsupported with facts.
- Threat: nullable conversion drops nulls.
  Mitigation: every nullable accepted shape has oracle null/value comparison and Arrow decode checks.
- Threat: unsupported conversion returns partial bytes.
  Mitigation: unsupported results return diagnostics and no `.loom` bytes.
</threat_model>
