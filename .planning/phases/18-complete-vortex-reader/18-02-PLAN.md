<objective>
Populate the complete-reader facts from real Vortex files by recursively inspecting root dtype, layout tree, child relationships, segment ids/ranges/alignment, statistics presence, and split ranges without emitting new `.loom` shapes yet.
</objective>

<context>
Plan 18-01 defines the fact model. Plan 18-02 fills those facts using `vortex-file 0.74.0` and `vortex-layout 0.74.0` APIs while preserving the Phase 15 narrow conversion path. This is the main evidence that Phase 18 is a real reader boundary rather than only a one-shape ingress shim.
</context>

<threat_model>
- Threat: traversal panics on unknown or complex layouts.
  Mitigation: traverse through public `Layout` APIs and convert errors to stable diagnostics.
- Threat: segment facts miss overlap/order hazards.
  Mitigation: compute Loom-owned segment ordering and overlap classifications from `SegmentSpec::byte_range()`.
- Threat: fact extraction performs excessive I/O.
  Mitigation: use metadata/layout/segment map APIs by default; avoid `display_tree_with_segments` unless a test explicitly opts in.
- Threat: scan/splits failures turn valid inspection into rejection.
  Mitigation: record split/scan availability separately from file-open acceptance.
</threat_model>
