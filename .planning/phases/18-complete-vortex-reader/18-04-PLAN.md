<objective>
Add real Vortex struct/table `.vortex` -> `LMT1` emission for files whose fields are all in the accepted single-column support matrix, with Phase 17 artifact verifier acceptance and Vortex scan oracle evidence.
</objective>

<context>
The final Loom path needs table-shaped artifacts, not just scalar columns. Phase 8 already introduced `LMT1` mixed-column payloads. Phase 18 should prove that real Vortex struct/table files can produce `LMT1` where every field is explicitly supported, while nested/unsupported fields fail closed.
</context>

<threat_model>
- Threat: table emission accepts unsupported child columns.
  Mitigation: table support is accepted only when every field passes the single-column support predicate.
- Threat: field order, names, row counts, or nulls drift during conversion.
  Mitigation: tests compare projected Loom rows/aggregates against Vortex scan oracle and table facts.
- Threat: nested structs/lists accidentally widen scope.
  Mitigation: nested shapes remain unsupported unless explicitly represented in the support matrix.
- Threat: CLI emission ambiguity between `LMC1` and `LMT1`.
  Mitigation: reader facts and CLI output include emission kind.
</threat_model>
