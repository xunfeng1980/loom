# Phase 43.2 Context: Production Native Codegen Stabilization and Production Readiness

## Why This Phase Exists

Phase 43.1 made an important leap: the Phase 35 Arrow semantic matrix can now
produce real MLIR/LLVM/JIT/native Arrow buffers, and those outputs are admitted
only after Phase 40 native/model validation. That is implementation evidence.

It is not yet enough evidence for ABI freeze. Phase 44 will freeze host/runtime
contracts that downstream code must live with. Freezing immediately after a
newly green JIT path risks locking an ABI around behavior that has not yet been
stressed for determinism, replay, production routing, failure injection,
resource ownership, observability, and cache drift.

Phase 43.2 is therefore a stabilization phase, not a feature expansion phase.
It must make the real native codegen path boring, diagnosable, repeatable, and
hard to accidentally bypass before Phase 44 freezes anything.

## Current Evidence From Phase 43.1

- `loom-native-melior` has a real `ExecutionEngine` entry point:
  `loom_arrow_semantic_codegen_buffers`.
- Primitive value buffers use typed memrefs for Int32, Int64, Float32, and
  Float64.
- Boolean value bitmaps and nullable validity bitmaps use byte memrefs.
- `loom-core` rebuilds Arrow arrays from JIT-produced buffers and runs Phase 40
  native/model validation before runtime/cache admission.
- Divergent output fails closed or takes explicit interpreter fallback according
  to policy; divergent output cannot seed native cache identity.
- The focused gate unsets `LOOM_ALLOW_NATIVE_TOOL_SKIP` and runs the real
  `--features melior` positive JIT test.

## The Remaining Production Gap

The remaining gap is not "does native codegen exist?" It does.

The gap is "can this path be frozen into an ABI without discovering later that
the real production contract needs different ownership, identity, diagnostics,
concurrency, cache, or fallback semantics?"

Phase 43.2 closes that gap with production-readiness evidence.

## Stabilization Dimensions

### Determinism and Replay

The same artifact, verifier facts, schema, projection, predicate, split,
backend identity, toolchain identity, and pipeline identity must produce stable
support fingerprints, output buffer digests, trace fingerprints, validation
records, and cache keys. Any drift must be visible before cache admission.

### Production Route Closure

Positive native-codegen evidence must come from the production runtime route, not
only test-local helper calls. The route must exercise fallback, cancellation,
missing toolchain, validation divergence, cache mismatch, and unsupported-shape
paths without emitting partial positive evidence.

### Adversarial Output Validation

The validator must be attacked at the actual Arrow buffer boundary: wrong byte
lengths, wrong validity bitmap lengths, wrong null counts, offset/bitmap drift,
schema mismatch, trace mismatch, malformed containers, direct `LMA1` bridge
inputs, zero-row batches, one-row batches, and multi-column nullable matrices.

### Resource and Performance Readiness

The native path must survive repeated execution and record basic production
signals: cache hit/miss behavior, cancellation latency, memory/lifetime
discipline, release ownership, and bounded native-vs-interpreter timing on
representative Phase 35 rows. This phase does not promise GA performance; it
collects enough data to avoid freezing an ABI blind.

### Diagnostics and ABI Pressure

Phase 44 needs a dossier of the real native-codegen contract: buffer ownership,
alignment, lifetime, backend/toolchain identity, pipeline fingerprints, trace
fingerprints, validation/fallback statuses, error codes, observability fields,
cache-key schema, concurrency/reentrancy assumptions, and explicit TCB
boundaries.

## Non-Claims

- Phase 43.2 does not prove MLIR/LLVM correctness.
- Phase 43.2 does not expand supported semantics beyond the Phase 35 matrix,
  except boundary and adversarial cases needed to stabilize that matrix.
- Phase 43.2 does not freeze the ABI.
- Phase 43.2 does not close suspended `ENGINE-01` live StarRocks runtime
  evidence.

## Gate Shape

The focused gate should become something like:

```sh
bash scripts/production-native-codegen-stabilization-test.sh
```

It should be strict by default:

- no accepted positive evidence from skipped native tooling;
- no accepted positive evidence from helper-only paths;
- no accepted cache identity without Phase 40 validation;
- no accepted route that uses `reference_zeroed_value_bytes`;
- no unversioned drift in diagnostics or cache identity fields targeted for
  Phase 44.
