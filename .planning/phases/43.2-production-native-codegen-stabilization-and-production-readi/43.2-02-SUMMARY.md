# Phase 43.2-02 Summary: Production Runtime Route Closure

**Status:** Complete
**Date:** 2026-06-09
**Plan:** `43.2-02-PLAN.md`

## What Changed

- Added `ArrowSemanticCodegenRouteStatus` and
  `ArrowSemanticCodegenProductionRouteReport` in `loom-native-melior`.
- Added `execute_arrow_semantic_codegen_production_route`, a production route
  that composes:
  - verifier-gated codegen support extraction;
  - real `execute_arrow_semantic_codegen_jit`;
  - Phase 40 output validation;
  - runtime decision;
  - replay/cache evidence admission.
- Added `validate_arrow_semantic_codegen_production_route_output`, which lets
  tests and future route callers validate backend-provided JIT output through
  the same production admission logic.
- Route reports now distinguish native candidate, interpreter fallback,
  fail-closed, and cancellation outcomes.
- Route evidence is cacheable only when replay evidence exists and the runtime
  decision is `NativeCandidate`.
- Added `production_arrow_semantic_route` tests covering:
  - real feature-enabled JIT route success;
  - output divergence fail-closed;
  - output divergence explicit interpreter fallback;
  - unsupported input fail-closed/fallback;
  - cancellation;
  - default non-feature `JitUnavailable` fail-closed behavior.

cargo test -p loom-native-melior --test production_arrow_semantic_route
cargo test -p loom-native-melior --features melior --test production_arrow_semantic_route
cargo test -p loom-native-melior --features melior --test production_arrow_semantic_jit

## Evidence

- Positive production route evidence comes from real JIT execution plus Phase 40
  validation, not helper-only output comparison.
- Unsupported shapes and missing default-toolchain feature paths cannot produce
  cacheable native evidence.
- Divergent JIT output cannot produce replay evidence or native cache admission.
- Fallback-enabled policy records interpreter fallback explicitly without making
  native evidence cacheable.
