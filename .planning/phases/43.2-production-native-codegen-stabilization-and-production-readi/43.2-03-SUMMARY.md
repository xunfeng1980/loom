# Phase 43.2-03 Summary: Adversarial Native Output Validation Matrix

**Status:** Complete
**Date:** 2026-06-09
**Plan:** `43.2-03-PLAN.md`

## What Changed

- Added `native_arrow_semantic_codegen_adversarial` tests covering malformed
  native output buffers, bitmap drift, schema/trace drift, malformed artifacts,
  row-count boundaries, and direct `LMA1` bridge identity.
- Strengthened codegen support extraction to normalize bitmap-backed Arrow
  buffers:
  - validity bitmaps are extracted from offset-normalized null validity bytes;
  - Boolean value bitmaps are rebuilt from logical row values, so sliced Arrow
    buffers do not leak physical offsets into support/cache evidence.
- Strengthened output reconstruction for zero-row fixed-width columns by using
  type-aligned empty Arrow buffers instead of unaligned empty byte buffers.
- Strengthened nullable output validation with explicit null-count checking for
  provided validity buffers.

## Coverage

- Malformed native outputs now fail closed for:
  - too-short and too-long value buffers;
  - too-short and too-long validity buffers;
  - missing validity buffers for nullable columns with nulls;
  - all-valid bitmaps for columns expected to contain nulls;
  - extra validity buffers for non-null outputs;
  - swapped or dropped columns.
- Boundary coverage now includes:
  - Boolean value bitmap corruption;
  - validity bitmap drift;
  - sliced buffers with non-zero offsets;
  - zero-row, one-row, all-null, nullable-no-null, and non-null primitive
    matrices.
- Artifact/trace coverage now includes:
  - malformed `LMC2`, malformed direct `LMA1`, and non-Loom bytes;
  - multi-batch payload rejection;
  - schema name drift, type drift, and value trace drift after support
    extraction;
  - distinct replay/cache evidence for default `LMC2(LMA1)` and explicit direct
    `LMA1` bridge inputs.

## Evidence

- No malformed native output can become Phase 40 validated.
- No malformed native output can seed runtime cache keys or replay evidence.
- Sliced Arrow inputs preserve logical rows and bitmap semantics before native
  admission.
- Zero-row fixed-width output now exercises the same validation path without
  relying on accidental buffer alignment.
