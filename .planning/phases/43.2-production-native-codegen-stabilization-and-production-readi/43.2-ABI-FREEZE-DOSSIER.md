# Phase 43.2 ABI-Freeze Dossier

**Date:** 2026-06-09
**Consumer:** Phase 44 — ABI Freeze and Compatibility Contract
**Scope:** Production native Arrow semantic codegen for the bounded Phase 35
matrix.

## Freeze Inputs

Phase 44 should freeze or explicitly version the following fields because they
now carry production-route evidence:

- Artifact identity:
  - artifact digest;
  - artifact kind: default `LMC2(LMA1)` versus explicit direct `LMA1`;
  - payload kind: Arrow semantic payload.
- Backend identity:
  - backend name: `loom-production-native-arrow-semantic-codegen`;
  - backend version: currently `phase43.1-production-codegen`;
  - melior/JIT identity string;
  - MLIR/LLVM lowering pipeline identity;
  - feature/toolchain marker, currently `feature=melior`.
- Output layout:
  - one output column per supported input column;
  - stable column index;
  - value buffer bytes;
  - optional validity buffer bytes;
  - Boolean values as packed bitmaps;
  - fixed-width primitive values as native Arrow value buffers;
  - nullable validity bitmaps with explicit null-count validation.
- Validation identity:
  - Phase 40 native/model validation status;
  - reference trace fingerprint;
  - native trace fingerprint;
  - output buffer fingerprint;
  - validation fingerprint;
  - replay fingerprint.
- Runtime/cache identity:
  - runtime ABI version;
  - artifact digest;
  - facts/support fingerprint;
  - production lowering fingerprint;
  - projection, predicate, split, and fallback policy;
  - stable id and canonical input.
- Query-shape admission:
  - production native codegen is native/cache eligible only for full projection,
    no predicate, and a full-scan split whose row count matches the validated
    execution row count;
  - projection, predicate, row-range split, or mismatched full-scan row counts
    fail closed or fall back and cannot produce replay/cache evidence until
    those semantics are implemented and validated.
- DuckDB internal native handoff:
  - default DuckDB native execution no longer uses LMC1 raw-copy test facts;
  - `LoomDuckDbNativeBuffer` carries builder id, Arrow type, row count, value
    buffer, and optional validity buffer for nullable primitive Arrow semantic
    outputs;
  - LMC1 raw/table artifacts remain interpreter/fail-closed inputs, not DuckDB
    native candidates.
- Route status:
  - native candidate;
  - interpreter fallback;
  - fail closed;
  - cancelled.
- Diagnostics:
- unsupported artifact/payload/type/batch shape;
  - unsupported query shape;
  - native output mismatch;
  - native/model trace mismatch;
  - JIT unavailable;
  - JIT symbol missing;
  - invalid backend artifact;
  - cancelled.
- Resource evidence:
  - route steps completed;
  - cancellation checkpoint;
  - output buffer ownership;
  - output buffer lifetime;
  - release assumption;
  - value/validity byte counts;
  - explicit `raw_pointer_identity_used = false`.

## Ownership And Lifetime

Current implementation-local ownership:

- Native route output buffers are owned Rust `Vec<u8>` values in route reports.
- Rebuilt Arrow arrays are owned by Rust `RecordBatch` values.
- No host-facing raw pointer release callback is exposed by Phase 43.2.
- Raw pointer values are not used in cache, replay, route, or resource identity.

Phase 44 decision:

- If the ABI exposes native output buffers across FFI, it must freeze explicit
  ownership transfer, release callback, alignment, and lifetime rules.
- If the ABI keeps native output internal to Rust, it should still expose stable
  route status, diagnostics, validation, cache, and resource evidence fields.

## Alignment

Phase 43.2 found and fixed a concrete zero-row fixed-width alignment edge:

- empty fixed-width native output buffers must be represented by type-aligned
  Arrow buffers, not arbitrary empty byte buffers;
- Boolean and validity bitmaps must be normalized to logical row offsets before
  support/cache/replay evidence is produced.

Phase 44 decision:

- ABI docs should state that value buffers must satisfy Arrow alignment for the
  declared data type.
- Validity and Boolean bitmaps should be specified as logical offset-zero
  buffers for the output slice under validation.

## Cancellation

Phase 43.2 exposes three deterministic cancellation checkpoints:

- `before-support`
- `before-jit`
- `before-validation`

Each checkpoint is non-cacheable and cannot produce replay evidence. Phase 44
should decide whether these exact checkpoints become ABI statuses or are mapped
to a smaller public cancellation status plus structured diagnostics.

## Concurrency And Reentrancy

Current evidence:

- Repeated route execution is deterministic over the focused matrix.
- Route reports own their buffers and evidence records independently.
- No shared mutable cache is implemented by Phase 43.2; cache behavior is
  simulated through replay/cache key evidence.

Phase 44 decision:

- If a production cache is exposed, the ABI must freeze cache hit/miss/mismatch
  reporting, cache lifetime, and thread-safety rules.
- Route report ownership must remain independent of host object lifetime unless
  an explicit shared cache/arena is introduced.

## Observability Fields

Minimum fields Phase 44 should preserve:

- route status;
- backend identity;
- artifact kind and digest;
- row/column counts;
- schema fingerprint;
- validation status;
- reference/native trace fingerprints;
- replay fingerprint;
- runtime cache stable id and canonical input;
- diagnostics code/path/message;
- resource evidence fields.

## Open Phase 44 Decisions

- Whether `loom_runtime.h` exposes production native-codegen route reports
  directly or a reduced compatibility view.
- Whether output buffers cross the C ABI or remain internal to Rust/Arrow
  `RecordBatch` construction.
- Whether cancellation checkpoints are public ABI enum values or diagnostic
  subpaths.
- Whether persistent cache admission is in ABI scope now or remains a future
  internal implementation detail.
- How to version backend/toolchain/pipeline identity when the MLIR/LLVM stack
  changes.

## TCB And Non-Claims

The following remain trusted or empirical, not proven:

- Rust compiler and standard library;
- arrow-rs buffer and array construction semantics;
- melior bindings;
- MLIR/LLVM lowering and JIT execution;
- CPU execution;
- any future host FFI boundary.

Phase 43.2 does not claim verified compilation, a persistent production cache,
general Arrow native support, nested/logical/string native support, or GA
performance. Native codegen is stabilized only for the bounded Phase 35 matrix.

`ENGINE-01` remains suspended as pre-GA live second-engine evidence. Phase 43.2
does not claim live StarRocks runtime execution.
