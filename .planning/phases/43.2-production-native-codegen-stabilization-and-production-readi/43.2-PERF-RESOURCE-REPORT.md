## Cache / Replay Behavior

The soak test executes the same artifact through the real route five times:

- first execution is treated as a simulated cache miss;
- the next four executions produce identical runtime cache stable id, canonical
  input, and replay fingerprint, and are treated as simulated hits;
- an intentional value drift produces distinct cache/replay evidence and cannot
  masquerade as the previous artifact.

This confirms deterministic replay evidence for unchanged inputs and cache-key
drift for changed artifacts. It does not claim a persistent production cache is
implemented in the route.

## Cancellation Evidence

The route now exposes deterministic cancellation checkpoints:

- `before-support`
- `before-jit`
- `before-validation`

Each checkpoint returns:

- `Cancelled` route status;
- non-cacheable route evidence;
- no replay evidence;
- a deterministic diagnostic path;
- route-step evidence showing which stages completed before cancellation.

This gives Phase 44 concrete status/diagnostic fields to freeze or map into the
host ABI.

## Ownership / Lifetime Evidence

Current production route evidence classifies native output buffers as:

- ownership: `owned-rust-vec`
- lifetime: `route-report-owned-before-host-abi-handoff`
- release assumption: dropped by the Rust owner; no raw pointer release callback
  is exposed yet
- raw pointer identity: not used in cache/replay/resource identity

Phase 44 ABI pressure:

- If host-facing native output crosses FFI, the ABI must add explicit buffer
  ownership, release callback, alignment, and lifetime rules.
- Cache/replay identity must remain based on artifact/backend/toolchain/trace
  fingerprints and output bytes, not raw addresses.
