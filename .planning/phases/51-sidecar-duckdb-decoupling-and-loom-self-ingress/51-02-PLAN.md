<objective>
Create the `loom-self-ingress` crate as the single IO boundary for `.loom` files, wrapping `loom-container` codecs. Feature-gate the `loom-cli` binary so the `sidecar embed` command compiles without `loom-container` (lean mode), while `inspect`/`decode`/`verify-artifact`/`ingest-vortex` require the `full` feature.

Purpose: Per RESEARCH.md, `loom-self-ingress` centralizes all `.loom`-format file access so no other crate imports `loom-container` directly for file IO (internal module use within `loom-core`'s verify/decode paths remains acceptable). The CLI split ensures `loom sidecar embed` — which only needs `loom-ir-core` + `loom-parquet-ingress` — can be built and distributed without the full container stack.

Output: `crates/loom-self-ingress/` crate with `.loom` file IO API; `loom-cli` with Cargo features enabling lean compilation.
</objective>

  <action>
Create `crates/loom-self-ingress/` as a library crate that wraps `loom-container` for file-system IO of `.loom` files.

**Cargo.toml:**
- `[dependencies]`: `loom-container = { path = "../loom-container" }`, `arrow = { workspace = true }` (for Arrow types in decode output — container decodes to Arrow)
- Description: "Single IO boundary for Loom native .loom file format — delegates all format logic to loom-container"

**src/lib.rs:** Provide three public functions:

1. `read_loom_file(path: &Path) -> Result<Vec<u8>, SelfIngressError>`
   - Read file bytes via `std::fs::read`
   - Validate container magic via `loom_container::container_codec::is_container_payload(&bytes)` — reject non-loom files with `NotALoomFile` error
   - Return raw bytes (callers decode further). Per RESEARCH.md Pattern 3, self-ingress does not re-implement codec logic; it provides the validated IO boundary.

2. `write_loom_file(path: &Path, bytes: &[u8]) -> Result<(), SelfIngressError>`
   - Validate input bytes are a valid container via `loom_container::container_codec::decode_container(bytes)` — reject malformed with `InvalidContainer` error
   - Write to disk via `std::fs::write`, using atomic temp-file + rename pattern (mirroring `sidecar_parquet.rs`'s approach): write to `{path}.tmp.loom-ingress`, then `std::fs::rename`
   - On IO error: return `SelfIngressError::Io(std::io::Error)`

3. `verify_loom_file(path: &Path) -> Result<ContainerDescription, SelfIngressError>`
   - Read and validate like `read_loom_file`
   - Return the decoded `ContainerDescription` so callers can inspect version/sections/features without re-decoding

**SelfIngressError enum:**
- `Io(std::io::Error)` — file read/write failure
- `NotALoomFile` — magic bytes don't match container
- `InvalidContainer(String)` — container decode failed with message

Implement `std::fmt::Display` and `std::error::Error` for `SelfIngressError`.

**Add to workspace:** Append `"crates/loom-self-ingress"` to root `Cargo.toml` members list.

After creation, verify: `cargo build -p loom-self-ingress` succeeds; `cargo tree -p loom-self-ingress | grep loom-container` confirms the dependency; `cargo doc -p loom-self-ingress --no-deps` confirms the three public functions are documented.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-06 | Tampering | `loom-self-ingress::read_loom_file` | mitigate | Validates container magic via `is_container_payload` before returning bytes; non-loom files rejected with `NotALoomFile` error |
| T-51-07 | Tampering | `loom-self-ingress::write_loom_file` | mitigate | Validates bytes via `decode_container` before writing; atomic temp-file + rename pattern prevents partial writes; malformed input returns `InvalidContainer` |
| T-51-08 | Information Disclosure | `loom-cli` feature gate bypass | mitigate | Feature gating is compile-time — Cargo `cfg(feature = "full")` cannot be bypassed at runtime; lean build binary physically lacks container-dependent code paths; verified by `cargo tree --no-default-features | grep loom-container` returning zero |
