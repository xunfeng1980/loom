<objective>
Create the lean `loom-sidecar-ffi` staticlib crate that exports sidecar extract/verify/routing/free functions via the C ABI, depending only on `loom-ir-core` and `loom-parquet-ingress` — zero transitive dependency on `loom-container`. Also remove `loom-core` from `loom-parquet-ingress` production dependencies so the lean path stays container-free.

Purpose: This is the first of three plans implementing Phase 51. Without this lean FFI surface, the DuckDB extension cannot access sidecar metadata without pulling in the full loom-container (19 modules, Arrow codecs, native lowering, etc.). The parquet-ingress dependency cleanup is a prerequisite for the lean path to avoid Cargo feature unification silently reintroducing `loom-container` via `loom-core` → `loom-container`.

Output: `crates/loom-sidecar-ffi/` crate with staticlib + rlib targets, `include/loom_sidecar.h` C header, and `loom-parquet-ingress` with `loom-core` in dev-dependencies only.
</objective>

Create `crates/loom-sidecar-ffi/` as a new `staticlib` + `rlib` crate following the existing `loom-ffi` patterns.

**Cargo.toml:**
- `[lib] crate-type = ["staticlib", "rlib"]`
- `[dependencies]`: `loom-ir-core = { path = "../loom-ir-core" }`, `loom-parquet-ingress = { path = "../../ingress/loom-parquet-ingress" }`, `parquet = { workspace = true }` (needed for `ParquetRecordBatchReaderBuilder` to open Parquet files and extract metadata)
- `[build-dependencies]`: `cbindgen = { workspace = true }`
- No `loom-core`, no `loom-container`, no `arrow` deps — sidecar FFI only deals with raw bytes, no Arrow types cross its C ABI boundary

**build.rs:** Mirror `crates/loom-ffi/build.rs` — create `include/` dir, read `cbindgen.toml`, run `cbindgen::Builder::new().with_crate().with_config().generate().write_to_file("include/loom_sidecar.h")`. Rerun-if-changed on `src/ffi.rs`, `src/lib.rs`, `cbindgen.toml`.

**cbindgen.toml:** Minimal — no Arrow FFI types to exclude since the sidecar FFI never references `FFI_ArrowArray`/`FFI_ArrowSchema`. Set `language = "C"`, `include_guard = "LOOM_SIDECAR_H"`, header comment identifying the file. No `after_includes` needed (no struct forward declarations). Exclude `LoomSidecarError` from the export list so cbindgen doesn't emit the Rust enum as a C struct.

**src/lib.rs:**
- `#![forbid(unsafe_code)]` at crate level (unsafe only in ffi.rs)
- `#[global_allocator] static GLOBAL: System = System;` — same allocator as loom-ffi, required for staticlib linking into C++ consumers (PITFALLS P5)
- `pub mod ffi;` module declaration

**src/ffi.rs:** `extern "C"` surface with four entry points, all wrapped in `catch_unwind(AssertUnwindSafe(...))`:

1. `loom_sidecar_extract(file_path: *const c_char, out_bytes: *mut *mut u8, out_len: *mut usize) -> i32`
   - Null-check all three pointer args → return NullPointer(1)
   - Convert `file_path` to `&Path`, open Parquet file via `parquet::arrow::arrow_reader::ParquetRecordBatchReaderBuilder`
   - Call `loom_parquet_ingress::sidecar_parquet::extract_sidecar_from_parquet_metadata(metadata)`
   - On `Ok(Some(overlay))`: encode via `overlay.encode()`, box the vec, forget the box, write ptr+len to out_params, return 0
   - On `Ok(None)`: return NoSidecar(5)
   - On `Err(e)`: return DecodeFailed(3)
   - On panic: return Panicked(4)

2. `loom_sidecar_verify(overlay_bytes: *const u8, overlay_len: usize, out_hash: *mut *const c_char) -> i32`
   - Null-check pointer args → return NullPointer(1)
   - Decode overlay: `SidecarOverlay::decode(&bytes)`
   - Compute hash: `l2core_codec::l2core_program_hash` over `overlay.ir_bytes`
   - Convert hash to CString, forget it, write ptr to out_hash
   - On malformed/truncated decode: return DecodeFailed(3)
   - On panic: return Panicked(4)

3. `loom_sidecar_route(overlay_bytes: *const u8, overlay_len: usize, host_data: *const u8, host_data_len: usize, out_decision: *mut *const c_char) -> i32`
   - Null-check all pointer args → return NullPointer(1)
   - Decode overlay from bytes
   - For each binding in overlay.bindings, call `verify_chunk_binding(&binding, host_data_slice)` — but only if host_data is provided and binding range is within bounds
   - Build `SidecarRoutingInput { engine_integrated: true, sidecar: Some(overlay), hash_verification: results, encoding_supported: true }`
   - Call `decide_sidecar_routing(input)`
   - Format decision as JSON string: `{"decision":"LoomNative"}` or `{"decision":"HostNativeReader","reason":"HashMismatch","diagnostics":[...]}`
   - Convert to CString, forget, write ptr to out_decision, return 0
   - On decode/route failure: return DecodeFailed(3)
   - On panic: return Panicked(4)

4. `loom_sidecar_free_bytes(ptr: *mut u8, len: usize) -> i32`
   - Null-check → NullPointer(1)
   - Reconstruct `Vec<u8>` from `ptr` and `len` via `unsafe { Vec::from_raw_parts(ptr, len, len) }` and drop it
   - Return 0

**LoomSidecarError enum** (module-private, `#[repr(i32)]`):
- NullPointer = 1, IoError = 2, DecodeFailed = 3, Panicked = 4, NoSidecar = 5

**Add to workspace:** Append `"crates/loom-sidecar-ffi"` to the `members` list in root `Cargo.toml`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| C ABI (loom-sidecar-ffi → DuckDB C++ extension) | Untrusted file paths and raw bytes cross from C++ into Rust via `extern "C"` entry points |
| File system (loom-sidecar-ffi → Parquet files) | User-provided file paths read by Parquet SDK; malformed content must fail closed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-01 | Tampering | `ffi.rs::loom_sidecar_verify` | mitigate | `SidecarOverlay::decode` performs bounds-checked parsing; malformed input returns `DecodeFailed(3)` — no undefined behavior on crafted bytes |
| T-51-02 | Denial of Service | `ffi.rs` all entry points | mitigate | Every `extern "C"` function body is wrapped in `std::panic::catch_unwind(AssertUnwindSafe(...))`; panics map to `Panicked(4)` return code, preventing unwind across C ABI |
| T-51-03 | Information Disclosure | `ffi.rs::loom_sidecar_extract` | mitigate | Null pointer checks on all three pointer args (`file_path`, `out_bytes`, `out_len`) before any dereference; returns `NullPointer(1)` on null |
| T-51-04 | Information Disclosure | `ffi.rs::loom_sidecar_extract` | accept | Path traversal is handled by the OS `open()` syscall; the sidecar FFI does not perform custom path resolution logic — file access is bounded by OS permissions |
| T-51-05 | Tampering | `lib.rs` global allocator | mitigate | `#[global_allocator] static GLOBAL: System = System;` ensures Rust heap and DuckDB's C++ heap share the same system allocator; cross-boundary free (via `loom_sidecar_free_bytes`) reconstructs Vec from ptr+len via `Vec::from_raw_parts`, which is sound because the allocation came from the same allocator |
| T-51-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages introduced; all deps are workspace-internal (`loom-ir-core`, `loom-parquet-ingress`) or already-audited workspace deps (`cbindgen`, `parquet`). slopcheck: not applicable — no new external packages. |
</threat_model>
