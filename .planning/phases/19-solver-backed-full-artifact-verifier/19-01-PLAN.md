<objective>
Define Loom-owned solver obligation, backend declaration, SMT-LIB script, and discharge report types without adding any solver dependency to `loom-core`.
</objective>

<context>
Phase 17 introduced `ConstraintDischargeStatus` but only records `CollectedOnly`. Phase 19 needs a richer solver evidence contract before any executable backend can safely update artifact facts.
</context>

<threat_model>
- Threat: solver evidence becomes tied to one solver binary.
  Mitigation: define backend declarations for `z3`, `cvc5`, and `bitwuzla` in `loom-core` while keeping execution outside `loom-core`.
- Threat: `Discharged` is set without complete evidence.
  Mitigation: model required obligations, per-obligation results, backend metadata, and aggregate status explicitly.
- Threat: Bitwuzla primary choice is lost during implementation.
  Mitigation: contract states Bitwuzla is the Phase 19 primary backend and `QF_BV` is the required path.
</threat_model>
