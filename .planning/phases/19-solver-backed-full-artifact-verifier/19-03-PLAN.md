<objective>
Add the optional subprocess solver crate with backend declarations for `z3`, `cvc5`, and `bitwuzla`, implementing Bitwuzla as the first full backend.
</objective>

<context>
Phase 19 intentionally keeps solver execution out of `loom-core`. This plan creates the execution boundary and makes Bitwuzla primary while keeping Z3/cvc5 as declared adapter kinds.
</context>

<threat_model>
- Threat: missing Bitwuzla is silently treated as solver proof.
  Mitigation: normal mode returns explicit skipped evidence; strict mode fails.
- Threat: stdout parsing accepts malformed output.
  Mitigation: parse only decisive `unsat`, `sat`, or `unknown` tokens and reject malformed output.
- Threat: backend trait becomes Bitwuzla-only.
  Mitigation: declare `z3`, `cvc5`, and `bitwuzla` backend kinds and expose discovery metadata for all three.
</threat_model>
