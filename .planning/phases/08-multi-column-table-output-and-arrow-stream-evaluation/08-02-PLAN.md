# Phase 08-02 Plan: Multi-Column Fixtures, Rust Decode, and CLI Table Output

## Purpose

Generate deterministic multi-column table fixtures and make Rust/CLI workflows decode them row-wise.

## Tasks

1. Add table fixture emitter.
   - Generate a deterministic table payload under `target/loom-duckdb-fixtures/`.
   - Include at least Int32, Boolean, and Utf8 columns.
   - Include manifest rows and aggregate expectations.
2. Add Rust table decode helper.
   - Decode each column via existing `decode_layout_to_array_data`.
   - Enforce shared row count.
   - Return typed column arrays or a simple table result suitable for CLI and DuckDB.
3. Extend CLI.
   - `loom inspect <table-payload>` prints table row count and column list.
   - `loom decode <table-payload>` prints row-wise output with headers.
   - Existing single-column CLI behavior remains stable.
4. Add tests.
   - table payload -> Rust decode;
   - CLI inspect/decode smoke for table payload;
   - existing descriptor/payload tests still pass.

## Acceptance Criteria

- A generated table payload contains multiple named columns of mixed types.
- CLI table decode prints row-wise data with column headers.
- Existing single-column CLI examples still pass.

## Verification

- `cargo test --workspace`
- `cargo run -p loom-fixtures --bin emit_duckdb_payloads`
- `cargo run --bin loom -- inspect target/loom-duckdb-fixtures/<table>.loom`
- `cargo run --bin loom -- decode target/loom-duckdb-fixtures/<table>.loom`
