# Phase 08-01 Plan: Table Model and Table Payload Codec

## Purpose

Add a table-shaped model and checked table payload codec while preserving compatibility with existing single-column payloads.

## Tasks

1. Add table data model in `loom-core`.
   - `TableDescription` with shared row count and ordered columns.
   - `TableColumn` with name, `LayoutDescription`, dtype, and nullable metadata if needed.
   - Validation for duplicate names, empty names, and row-count mismatch.
2. Add checked table payload codec.
   - Use a new magic/version, for example `LMT1`, so it is distinct from single-column `LMP1`.
   - Encode ordered column names and per-column layout payloads or equivalent structured data.
   - Decode with typed errors, no panics.
3. Preserve single-column compatibility.
   - Do not change existing `decode_layout_payload`.
   - Existing `.loom` payloads remain accepted by CLI and DuckDB.
4. Add unit tests.
   - table payload roundtrip;
   - duplicate/empty column names rejected;
   - mismatched row counts rejected;
   - mixed Int32/Boolean/Utf8 columns represented.

## Acceptance Criteria

- `cargo test -p loom-core table` passes.
- Existing `cargo test -p loom-core layout_codec` still passes.
- `scripts/mvp0-verify.sh` remains green.
