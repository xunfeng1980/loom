<objective>
Promote the existing melior validation and LLVM lowering code into a production
backend pipeline that records backend identity and fails closed.
</objective>

<context>
`loom-native-melior` already validates production-standard MLIR and checks
translation to LLVM IR. Phase 23 needs to connect that evidence to the backend
request/report model and make the pass pipeline identity part of the artifact.
</context>

<threat_model>
- Threat: MLIR validation succeeds for an artifact that runtime policy rejected.
  Mitigation: pipeline entry points require a validated `NativeBackendRequest`.
- Threat: pass-pipeline changes invalidate cache identity silently.
  Mitigation: include pipeline string/hash in `NativeBackendIdentity` and reports.
- Threat: toolchain mismatch changes behavior.
  Mitigation: record LLVM/MLIR version facts and target/layout information where
  available.
</threat_model>
