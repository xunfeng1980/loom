<objective>
Add a narrow verifier-gated LLVM/JIT execution seed for supported primitive
kernels and compare it against interpreter output.
</objective>

<context>
Phase 16 proved a bounded JIT boundary. Phase 23 should make the production
backend capable of executing only supported native-candidate plans after runtime
policy, verifier facts, production lowering, and backend pipeline validation all
accept.
</context>

<threat_model>
- Threat: JIT runs before verifier/runtime policy acceptance.
  Mitigation: JIT entry points accept only prepared backend artifacts from 23-03.
- Threat: unsupported encodings silently fall back inside the backend.
  Mitigation: backend reports distinguish unsupported/interpreter-policy from JIT
  execution and do not make host fallback decisions.
- Threat: native output diverges from interpreter output.
  Mitigation: add focused equivalence tests over the supported primitive matrix.
</threat_model>
