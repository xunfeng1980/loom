<objective>
Degrade the three existing ingress crates (Vortex, Parquet, Lance) into thin host adapters. Each adapter's public API becomes: mount host file → extract source facts + sidecar stubs + hash-binding stubs. All Arrow materialization and LMC2/LMA1 emission code is removed from public API and either deleted or moved to `#[cfg(test)]` modules.

Purpose: Fulfill Repositioning Decision Two (§8 item 4): "three hosts = one IR + three thin adapters." The adapters must NOT decode, materialize Arrow, or emit containers — those are host-native-reader responsibilities and the sidecar overlay (Phase 50). This phase removes the old emission path; the new sidecar contract is Phase 50.
Output: Three degraded ingress crates with slim public APIs (fact extraction + sidecar/hash stubs); all emission code removed from public surface.
</objective>

| Boundary | Description |

| cfg(test)-gated oracle functions | Arrow materialization only available in test contexts; cannot be called from production code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50.1-04 | Denial of Service | extract_sidecar_bytes stubs | mitigate | Stubs return `Option<Vec<u8>>` with `None` = "no sidecar present — fall back to host reader"; downstream must check return value before assuming overlay exists |
| T-50.1-05 | Elevation of Privilege | cfg(test)-gated functions | mitigate | `#[cfg(test)]` prevents oracle batch functions from being called in production; Rust compiler enforces this at crate boundary |
| T-50.1-06 | Information Disclosure | Ingress adapter public API | accept | No new sensitive data exposure — adapters expose only source-neutral facts (already public before this phase) |
| T-50.1-07 | Tampering | bind_content_hash stubs | accept | Stubs are no-ops returning Ok(()); real implementation in Phase 50; no production impact until sidecar overlay exists |
| T-50.1-SC | Tampering | cargo installs | accept | No new packages installed in this phase |
