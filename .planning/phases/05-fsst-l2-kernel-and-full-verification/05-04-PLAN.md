<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Payload file path -> DuckDB extension | SQL caller supplies the path string read by the extension. |
| Payload bytes -> Rust FFI | Payload bytes are untrusted input to `loom_decode`. |
| Arrow C buffers -> DuckDB vectors | C++ reads Arrow buffers and copies/sets DuckDB-owned vector values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Tampering | payload path | mitigate | Treat read failures as DuckDB `IOException`; do not silently fall back to legacy stub for non-empty paths. |
| T-05-13 | Denial of Service | Arrow buffer population | mitigate | Check buffer pointers and `n_buffers` for each supported type before reading. |
| T-05-14 | Information Disclosure | Utf8 strings | mitigate | Copy strings into DuckDB-owned storage; do not retain pointers into released Arrow buffers. |
| T-05-15 | Repudiation | SQL acceptance | mitigate | Script checks exact rows/aggregates for every supported encoding and exits nonzero on mismatch. |

No high open threats remain if final verification passes.
</threat_model>
