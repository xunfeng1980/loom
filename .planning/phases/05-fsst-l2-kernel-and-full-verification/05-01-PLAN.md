<objective>
Implement the core FSST L2 kernel and Utf8-aware L1/L2 integration in `loom-core`.
This plan replaces the Phase 04 empty Utf8 FSST stub with real `fsst-rs` decode,
adds a validated Loom-owned params format, extends Arrow builder output to Utf8,
and lets `Dictionary { values: KernelEscape(FSST) }` decode through the general
dictionary path.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FSST params bytes -> loom-core | Params may be malformed or malicious and must be validated before indexing, allocation, or decompression. |
| fsst-rs decoder -> FsstKernel | `fsst-rs` can assert on malformed compressed streams; the kernel must convert normal malformed input into typed errors. |
| Dictionary codes -> Utf8 values | Codes can be null, negative, or out of range; values can be null. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of Service | FSST params parser | mitigate | Validate lengths with checked arithmetic and reject truncated params before slicing. |
| T-05-02 | Denial of Service | FSST offsets | mitigate | Require `codes_offsets.len() == row_count + 1`, monotonic offsets, and last offset <= codes bytes length. |
| T-05-03 | Denial of Service | fsst-rs assertions | mitigate | Prevalidate params and convert any decoder panic into a typed `LoomDecodeError`. |
| T-05-04 | Tampering | Utf8 output | mitigate | Verify decoded bytes with `std::str::from_utf8` before appending to Arrow StringBuilder. |
| T-05-05 | Tampering | dependency boundary | mitigate | Run `cargo tree -p loom-core | grep -c -E 'vortex|fastlanes'` and require `0`. |

No high open threats remain if all mitigations are implemented and verified.
</threat_model>
