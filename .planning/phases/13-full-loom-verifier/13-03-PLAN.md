## Trust Boundaries

| Boundary | Description |
|---|---|
| L2Core artifact -> verifier | Malformed programs must produce diagnostics before facts are emitted. |
| Verifier -> diagnostics | Rejections must be stable enough for tests and reviewer reports. |
| Verifier -> lowering facts | Facts must only be emitted for accepted programs. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-13-07 | Elevation of privilege | input capabilities | mitigate | Missing capability tests must fail. |
| T-13-08 | Tampering | output builder state | mitigate | Type/effect tests reject mismatched Arrow event types. |
| T-13-09 | Denial of service | cursor loops | mitigate | Non-monotone cursor loops fail through `Decreases`/progress diagnostics. |
| T-13-10 | Repudiation | lowering facts | mitigate | Tests assert facts only exist for accepted programs. |

No high open threats remain if positive/negative verifier tests and CLI/internal evidence pass.
</threat_model>
