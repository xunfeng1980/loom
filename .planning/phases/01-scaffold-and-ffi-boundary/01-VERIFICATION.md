### CR-01 Review Finding — Confirmed Fixed

The REVIEW.md identified a blocker-class defect: `panic = "abort"` rendered the `catch_unwind` wrapper a no-op in the released staticlib, defeating DUCK-04. The fix (Option B from the review) was applied:

- **Before:** `Cargo.toml` had `panic = "abort"` — `catch_unwind` could not intercept panics in release builds.
- **After:** `Cargo.toml:50` has `panic = "unwind"` — `catch_unwind` is live.
- **Regression guard:** Invariant script CORE-02 block now asserts `panic = "unwind"` is present AND `panic = "abort"` is absent. CI workflow adds `cargo test -p loom-ffi --release` as a separate step.
- **Test confirmation:** `panic_does_not_abort` passes under `cargo test -p loom-ffi --release` — the catch_unwind intercepts the panic and returns `LoomError::Panicked` (code 3) without aborting.

**WR-01 (loom.h forward declarations):** Fixed. `cbindgen.toml:39` adds `after_includes` with `typedef struct ArrowArray FFI_ArrowArray; typedef struct ArrowSchema FFI_ArrowSchema;` — header is now self-contained.

**WR-02 (invariant script CORE-01 silent pass on cargo failure):** Fixed. Script now captures `cargo tree -d` exit code separately; empty arrow result fails (not passes); fail-closed behavior confirmed.
