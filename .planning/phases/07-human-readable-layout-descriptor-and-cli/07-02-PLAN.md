# Phase 07-02 Plan: Payload Inspection Bridge and Descriptor Fixtures

## Purpose

Connect existing binary `.loom` payloads to the descriptor format and prove descriptor roundtrips against real generated MVP0 fixtures.

## Tasks

1. Add a payload-to-descriptor bridge:
   - read existing LMP1 `.loom` bytes;
   - decode into `LayoutDescription`;
   - print descriptor text;
   - preserve current binary payload decode behavior.
2. Add fixture tests in `loom-fixtures`:
   - generate or reuse bitpack, FOR, dict, RLE, FSST, and dict-over-FSST fixtures;
   - convert fixture layouts to descriptor text;
   - parse descriptor text back;
   - decode both original and parsed layouts;
   - compare values/nulls against the Vortex oracle.
3. Start expanding sample coverage:
   - add at least one additional sample for two representative encodings;
   - prefer one numeric nullable sample and one UTF-8/FSST edge sample.
4. Keep `loom-core` Vortex-free.

## Acceptance Criteria

- Every MVP0 fixture can be printed as descriptor text and parsed back.
- Parsed descriptor decode matches the original fixture output and Vortex oracle.
- Existing `.loom` payload decode path remains unchanged.
- `scripts/mvp0-verify.sh` remains green.

## Verification

- `cargo test -p loom-fixtures descriptor`
- `cargo test --workspace`
- `bash scripts/mvp0-verify.sh`
