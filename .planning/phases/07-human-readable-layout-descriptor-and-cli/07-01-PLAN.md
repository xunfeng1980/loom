# Phase 07-01 Plan: Descriptor Format and Core Roundtrip

## Purpose

Define and implement the human-readable descriptor format for MVP0 layouts.

## Tasks

1. Inspect current `LayoutDescription`, `LayoutNode`, `FsstParams`, and `layout_codec` structures.
2. Decide descriptor format:
   - prefer RON or S-expression for recursive trees;
   - document why the chosen format fits nested layouts better than TOML;
   - keep the format MVP0-scoped and explicitly unstable.
3. Add descriptor module in `loom-core` or an adjacent crate-local module:
   - `to_descriptor_text(&LayoutDescription) -> Result<String, LoomError>` or equivalent;
   - `from_descriptor_text(&str) -> Result<LayoutDescription, LoomError>` or equivalent.
4. Cover all MVP0 node kinds:
   - Raw;
   - BitPack;
   - FrameOfReference;
   - Dictionary;
   - RunEnd;
   - KernelEscape with FSST params.
5. Add unit tests:
   - parse/print/parse roundtrip for each node kind;
   - nested dict-over-FSST layout;
   - invalid syntax returns a typed error, not a panic.

## Acceptance Criteria

- Descriptor text roundtrips deterministically for all MVP0 layout nodes.
- Parser errors are typed and do not panic.
- `cargo test -p loom-core descriptor` passes.
- `cargo tree -p loom-core | awk '/vortex|fastlanes/{c++} END{print c+0}'` prints `0`.
