# Phase 07-03 Plan: CLI Inspect and Decode

## Purpose

Expose reviewer-friendly commands for inspecting and decoding Loom payloads or descriptors without writing Rust tests.

## Tasks

1. Add a CLI binary target.
   - Prefer a small Rust binary in an existing crate unless a new crate is clearly cleaner.
   - Use minimal dependencies; avoid a large CLI framework unless already present.
2. Implement `loom inspect <input>`.
   - Accept binary `.loom` payloads.
   - If descriptor files are supported directly, detect by extension or parsing fallback.
   - Print dtype, row count, layout tree, kernel ids, and whether validity/nulls are present.
3. Implement `loom decode <input>`.
   - Print row values in a stable text or CSV-like format.
   - Represent nulls explicitly as `NULL`.
   - Support Int32, Int64, Boolean, and Utf8 MVP0 outputs.
4. Add CLI tests or integration smoke tests:
   - inspect generated `bitpack-i32.loom`;
   - decode generated `fsst-utf8.loom`;
   - invalid input exits nonzero with a useful message.
5. Document invocation in README.

## Acceptance Criteria

- `cargo run --bin loom -- inspect <payload>` prints a readable layout summary.
- `cargo run --bin loom -- decode <payload>` prints expected values/nulls for numeric and string payloads.
- CLI errors do not panic.
- `scripts/mvp0-verify.sh` remains green.
