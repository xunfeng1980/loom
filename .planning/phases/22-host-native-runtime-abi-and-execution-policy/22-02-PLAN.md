<objective>
Make native, interpreter, and fail-closed execution decisions deterministic from
verified artifact facts and host policy.
</objective>

<context>
The runtime must consume accepted artifact verification, solver discharge,
production lowering support, and reader/lowering disposition. It must reject or
fall back before any native plan can be prepared.
</context>

<threat_model>
- Threat: native planning accepts collected-only or unknown constraints.
  Mitigation: native requires `Discharged` or `NotRequired`.
- Threat: interpreter fallback silently hides unsupported semantics.
  Mitigation: fallback is explicit and policy-controlled.
- Threat: valid reader facts are confused with native-lowering support.
  Mitigation: runtime decision consumes lowering disposition separately.
</threat_model>
