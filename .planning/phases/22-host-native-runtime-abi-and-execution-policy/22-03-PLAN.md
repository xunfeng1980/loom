<objective>
Define the host-neutral query-shape envelope for projection, predicate pushdown,
splits, and worker-local concurrency.
</objective>

<context>
DuckDB exposes projection pushdown during table-function init, while StarRocks
will require columnar, splittable, pushdown-aware execution later. Phase 22 must
model these concerns before either host integration begins.
</context>

<threat_model>
- Threat: projection pushdown changes output column order or schema silently.
  Mitigation: projected output order is explicit and validated.
- Threat: unsupported predicate pushdown drops rows incorrectly.
  Mitigation: unsupported predicates either scan all rows or fail closed by
  policy.
- Threat: concurrent scan workers share mutable unsafe state.
  Mitigation: separate scan-global immutable state from worker-local state and
  require splittability declarations.
</threat_model>
