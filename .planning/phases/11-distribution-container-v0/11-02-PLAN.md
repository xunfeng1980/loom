<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Container header/features -> verifier | Unsupported required features must stop before payload decode. |
| Container payload -> raw layout/table decoders | Wrapped bytes must be extracted exactly once and then checked by existing decoders. |
| FFI caller -> container decode | Malformed containers must map to error codes without unwinding across C. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-05 | Tampering | wrapped payload section | mitigate | Verify section bounds and delegate to existing checked `LMP1`/`LMT1` decoders. |
| T-11-06 | Repudiation | diagnostics | mitigate | Stable verifier diagnostics for feature and section failures. |
| T-11-07 | Denial of Service | FFI malformed input | mitigate | FFI returns existing error codes; no C ABI changes or panics. |

No high open threats remain if container failures are visible through verifier and FFI decode errors.
</threat_model>
