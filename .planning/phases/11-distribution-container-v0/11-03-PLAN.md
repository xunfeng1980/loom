<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Container bytes -> CLI output | CLI must summarize sections without large byte dumps or misleading raw-payload output. |
| Container bytes -> DuckDB bind | C++ bind must parse only enough to derive schema/payload kind and reject malformed directory shape. |
| Generated fixtures -> release gate | Fixture generation should prove the new container path without erasing legacy compatibility tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-08 | Spoofing | DuckDB bind payload kind | mitigate | Distinct `LMC1` handling and extraction tests through smoke fixtures. |
| T-11-09 | Information disclosure | CLI section display | mitigate | Print concise header/section summaries, not section byte dumps. |
| T-11-10 | Tampering | generated fixture status | mitigate | Smoke test checks generated files use `LMC1`. |

No high open threats remain if CLI and DuckDB both recognize the same wrapped payload kind.
</threat_model>
