## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LayoutNode -> loom-core | LayoutNode fields may be malformed; dictionary codes, run ends, and kernel ids must be validated before indexing or dispatch |
| KernelEscape -> L2 kernel | Kernel-owned `ArrayData` crosses back into the same Arrow FFI-compatible output path as builder-owned arrays |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | dictionary code lookup | mitigate | Check non-null codes are non-negative and less than decoded values length; return typed invalid-code error before indexing |
| T-04-02 | Denial of Service | RunEnd expansion | mitigate | Validate run ends are strictly increasing, do not exceed `count`, and enough values exist before expansion |
| T-04-03 | Denial of Service | KernelEscape dispatch | mitigate | `L2KernelRegistry::get` returns Option; unknown ids return `LoomDecodeError::UnknownKernel`, never panic |
| T-04-04 | Tampering | Vortex isolation | mitigate | Keep `loom-core` dependency tree free of `vortex-*` and `fastlanes`; enforce with cargo tree check |

No high open threats remain if the acceptance checks pass.
</threat_model>
