<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vortex ArrayRef -> vortex_reader | Fixture bridge reads Vortex dict/RLE internals and converts them to plain LayoutNode data |
| vortex_reader -> loom-core | loom-core must receive only plain Rust/Arrow-independent data and keep zero Vortex dependencies |
| Test oracle -> phase acceptance | Oracle comparison must check both values and null positions row-for-row |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | dict/RLE bridge validity | mitigate | Compare Loom null flags against Vortex oracle null flags for nullable dict/RLE cases |
| T-04-06 | Repudiation | RLE representation mismatch | mitigate | If FastLanes RLE cannot be mapped exactly to Loom RunEnd in this phase, document the fallback in code and tests so the gap is visible, not silently accepted |
| T-04-07 | Tampering | loom-core dependency isolation | mitigate | Run `cargo tree -p loom-core | grep -c -E 'vortex|fastlanes'` and require `0` |

No high open threats remain if the fallback, if used, is explicit and tested.
