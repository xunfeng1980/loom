# Quick Task vl5: Distinguish corpus-level lineage evidence from per-artifact evidence

## Origin

MVP1.5 — Verified Lineage audit finding (medium, non-blocking).

In `crates/loom-core/src/verified_lineage.rs`, the three milestone-level evidence
layers were stamped with the per-artifact status `Passed`:

- `LeanModeledSoundnessTheorem`
- `LeanRustVerifierDifferential`
- `ModelRustInterpreterDifferential`

These are corpus/gate-level properties trusted-by-reference (the release gate
`scripts/verified-lineage-test.sh` ran them over the full fixture+fuzz corpus),
not facts re-validated when an individual artifact record is built. Stamping them
`Passed` reads as if each artifact was individually re-validated — the exact
overclaim shape the milestone red line polices.

## Change

1. Add `VerifiedLineageEvidenceStatus::CorpusValidated` (`as_str` → `"corpus-validated"`):
   "true for the full release corpus via the CI gate; trusted-by-reference at
   artifact granularity, not re-validated per artifact."
2. Use `CorpusValidated` for the three differential/soundness layers above, in
   place of `Passed`.
3. Leave unchanged:
   - `RustVerifierStructuralCheck` → `Passed` (genuinely per-artifact).
   - `BitwuzlaSmtDischarge` status logic (tracks `constraint_status`).
   - `NativeModelValidation` per-run logic (`PerRunValidated` / `NotRun` / fail-closed).
   - `LeanModeledSoundnessTheorem` `NotApplicable` branch (no `l2_core` facts).
4. Update `crates/loom-core/tests/verified_lineage.rs` assertions.
5. Add a `corpus-validated` row to the Phase 36 contract evidence-status table.

## Non-goals

- No change to the gate scripts, the Lean proofs, or any evidence layer's
  underlying check. This is a claim-granularity wording fix only.
- Does not touch the `verified-lineage-test.sh` grep markers (all preserved).

## Verification

- `cargo test -p loom-core --test verified_lineage` passes.
- `bash scripts/verified-lineage-test.sh` still passes (markers intact).

## Threat Model

- T-vl5-01 (overclaim): mitigated — record now distinguishes corpus-level
  trust-by-reference from per-artifact verification.
- T-vl5-02 (CI marker contract): mitigated — no existing grep marker removed.
