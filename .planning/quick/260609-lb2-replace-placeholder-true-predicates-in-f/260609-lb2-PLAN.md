  <name>Task 1: Add lookup helpers and mutual recursive checkers</name>
  <files>formal/lean/LoomCore.lean</files>
  <action>
Below the existing AST/`structure Program` block and ABOVE the current `finite_bounds`
definition, add the lookup helpers and three independent `Bool`-valued checkers exactly as
compiled in RESEARCH.md ("Concrete Lean 4 implementation"). Keep the file Lean-core only — no
Mathlib import, no lakefile reference.

Add two lookup helpers over the existing `List Capability`:
- `builderInfo? (caps : List Capability) (name : String) : Option (L2Ty × Bool)` via
  `caps.findSome?`, matching `Capability.outputBuilder id ty nullable _` and returning
  `(ty, nullable)` when `id == name`. Mirrors Rust `MissingOutputBuilder` / nullability lookup.
- `inputSlice? (caps : List Capability) (name : String) : Option (Nat × Nat)` via
  `caps.findSome?`, matching `Capability.inputSlice id offset length` and returning
  `(offset, length)` when `id == name`. Mirrors Rust `input_capabilities` lookup.

Define THREE independent checkers (per RESEARCH "Open Question 2" recommendation — 1:1 mapping
to Rust check groups), each a `mutual` block pairing `check<X>Stmt : ... -> Stmt -> Bool` with
`check<X>Body : ... -> List Stmt -> Bool`. The body helper recurses on the tail and into the
stmt helper for the head — this is the structural-recursion shape Lean 4.30 accepts with no
`termination_by`. Do NOT recurse a stmt checker directly over `List Stmt`.

- `checkTyped (caps : List Capability) : List Stmt -> Bool` (builder_events_typed):
  - `appendValue builder ty` -> `builderInfo? caps builder` returns `some (bty, _)` and `bty == ty`
    (OutputTypeMismatch + MissingOutputBuilder); `none` -> `false`.
  - `appendNull builder` -> `builderInfo? caps builder` returns `some (_, nullable)` and
    `nullable` is true (OutputNullabilityMismatch + MissingOutputBuilder); `none` -> `false`.
  - `forRange`/`cursorLoop` -> recurse into body via `checkTypedBody`; other arms (`readInput`,
    `failClosed`) -> `true`.
- `checkAuthority (caps : List Capability) : List Stmt -> Bool` (no_ambient_authority):
  - `readInput cap offset width _` -> `inputSlice? caps cap` returns `some (sOff, sLen)`
    (MissingInputCapability) and `decide (offset >= sOff) && decide (offset + width <= sOff + sLen)`
    (read spatial bounds, faithful because Lean offset/width are concrete `Nat`); `none` -> `false`.
  - `appendValue builder _` / `appendNull builder` -> require `builderInfo? caps builder` is
    `some` (shared MissingOutputBuilder); `none` -> `false`.
  - `forRange`/`cursorLoop` -> recurse via `checkAuthorityBody`; `failClosed` -> `true`.
- `checkBounds (caps : List Capability) (maxRows : Nat) : List Stmt -> Bool` (finite_bounds):
  - `forRange _ start stop body` -> `decide (stop >= start) && decide (stop - start <= maxRows)`
    (InvalidLoopBounds + ResourceBudgetExceeded) `&& checkBoundsBody caps maxRows body`.
  - `cursorLoop _ limit progress body` -> `decide (progress > 0) && decide (limit <= maxRows)`
    (NonMonotoneCursorLoop + ResourceBudgetExceeded) `&& checkBoundsBody caps maxRows body`.
  - other arms (`readInput`, `appendValue`, `appendNull`, `failClosed`) -> `true`.

Use `==` (BEq) for `String`, `decide (a <= b)` / `decide (a > b)` for `Nat` comparisons in a
`Bool` context, and `&&` for `Bool` AND. `L2Ty` already derives `DecidableEq` so `bty == ty`
works; the lookups pattern-match and do not need `DecidableEq` on `Capability`.
  </action>
